Unit 7: Digital Systems Design

7.1 Arithmetic Logic Unit (Design of 4-bit ALU)

The Arithmetic Logic Unit (ALU) is a core component of a processor that performs arithmetic and logical operations on binary data.

  • Definition: A 4-bit ALU processes 4-bit operands, performing operations like addition, subtraction, AND, OR, and NOT based on control signals.
  • Design of 4-bit ALU:
    • Inputs: Two 4-bit operands (A = A₃A₂A₁A₀, B = B₃B₂B₁B₀), control signals (S₁, S₀) to select operation, and a carry-in (C_in).
    • Outputs: 4-bit result (F = F₃F₂F₁F₀), carry-out (C_out), and status flags (e.g., zero, overflow).
    • Operations: Example operations based on control signals:
      S₁S₀Operation
      00F = A AND B
      01F = A OR B
      10F = A + B
      11F = A - B
    • Implementation: Uses four 1-bit ALUs in parallel, each with:
      • Logic unit: AND, OR gates for logical operations.
      • Arithmetic unit: Full adder for addition; subtraction via two’s complement (A - B = A + (B’ + 1)).
      • Multiplexer: Selects output based on S₁, S₀.
      Example: For addition, each 1-bit ALU computes Fᵢ = Aᵢ ⊕ Bᵢ ⊕ C_in, C_out = (Aᵢ · Bᵢ) + (Bᵢ · C_in) + (Aᵢ · C_in), chained via ripple carry.
  • Example: For A = 1010 (10₁₀), B = 0011 (3₁₀), S₁=1, S₀=0 (addition), result F = 1010 + 0011 = 1101 (13₁₀).
  • Applications: Used in CPUs for arithmetic (e.g., addition in loops), logical operations (e.g., bit manipulation), and data processing.

7.2 Accumulator

An accumulator is a register that stores intermediate results during processing, typically used in arithmetic and logic operations.

  • Definition: A special-purpose register that holds one operand and the result of an ALU operation, updated after each operation.
  • Operation: The accumulator receives data from memory or input, feeds it to the ALU, and stores the ALU’s output. Example: In ADD B, accumulator (A) adds B and stores A + B.
  • Structure: Typically a 4-bit or 8-bit register (e.g., D flip-flops). Example: A 4-bit accumulator uses four D flip-flops to store a 4-bit value, updated on clock pulses.
  • Example: Accumulator = 0011 (3₁₀), ADD 0100 (4₁₀). ALU computes 0011 + 0100 = 0111 (7₁₀), stored back in the accumulator.
  • Applications: Used in CPUs for iterative computations, data accumulation (e.g., summing values), and temporary storage in microprocessors.

7.3 Status Register and Flags

The status register, also called the flag register, stores condition flags that indicate the outcome of ALU operations.

  • Definition: A register containing single-bit flags that reflect the status of the processor after an operation.
  • Common Flags:
    • Zero Flag (Z): Set to 1 if the ALU result is zero. Example: 5 - 5 = 0 sets Z=1.
    • Carry Flag (C): Set to 1 if there’s a carry-out in addition or borrow in subtraction. Example: 15 + 1 = 16 (4-bit overflow) sets C=1.
    • Sign Flag (S): Set to 1 if the result is negative (MSB = 1 in two’s complement). Example: -3 (1101₂) sets S=1.
    • Overflow Flag (V): Set to 1 if arithmetic overflow occurs. Example: Adding two positive 4-bit numbers 7 (0111) + 7 (0111) = 14 (1110) sets V=1 (result appears negative).
  • Implementation: A 4-bit status register may store Z, C, S, V as individual flip-flops, updated after each ALU operation.
  • Example: For 7 - 4 = 3 (0111 - 0100 = 0011), flags: Z=0 (non-zero), C=0 (no borrow), S=0 (positive), V=0 (no overflow).
  • Applications: Flags control program flow (e.g., conditional jumps), indicate errors, and support decision-making in CPUs.

7.4 Processor Unit

The processor unit, or Central Processing Unit (CPU), is the core of a digital system, executing instructions by processing data through its components.

  • Definition: The processor unit fetches, decodes, and executes instructions, coordinating data flow between memory, ALU, and registers.
  • Components:
    • ALU: Performs arithmetic and logical operations (see 7.1).
    • Registers: Include accumulator, program counter (PC), instruction register (IR), and general-purpose registers for temporary storage.
    • Control Unit: Generates control signals to coordinate operations (e.g., fetch, decode, execute cycles).
    • Status Register: Stores flags to guide instruction execution (see 7.3).
  • Operation:
    • Fetch: PC retrieves instruction from memory, increments PC.
    • Decode: Control unit interprets instruction in IR.
    • Execute: ALU processes data, updates accumulator and flags.
    Example: For ADD B, fetch instruction, decode to add B to accumulator, execute via ALU, store result in accumulator, update flags.
  • Example: A 4-bit processor adds 1010 and 0011. PC fetches ADD instruction, ALU computes 1101, stores in accumulator, sets flags (e.g., Z=0, C=0).
  • Applications: CPUs in computers, microcontrollers, and embedded systems for executing programs and processing data.
