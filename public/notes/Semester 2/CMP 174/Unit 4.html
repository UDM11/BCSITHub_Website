Unit 4: Combinational Logic

4.1 Introduction to Combinational Logic Circuit

Combinational logic circuits produce outputs that depend solely on the current inputs, with no memory or feedback. They are fundamental building blocks in digital systems.

  • Definition: A combinational circuit’s output is a function of its inputs at any given time. Example: Adders, multiplexers, and decoders.
  • Characteristics: No memory elements; outputs are determined by Boolean functions of inputs. Example: A circuit computing F = A · B + C'.
  • Design Process: Define inputs/outputs, derive truth table, obtain Boolean expression, simplify using K-maps, and implement with logic gates.
  • Applications: Arithmetic operations, data routing, and code conversion in processors, calculators, and communication systems.

4.2 Adders and Subtractors

Adders and subtractors are combinational circuits used for binary arithmetic operations.

  • Half Adder: Adds two single-bit inputs (A, B). Outputs: Sum (S = A ⊕ B) and Carry (C = A · B).
    ABSumCarry
    0000
    0110
    1010
    1101
  • Full Adder: Adds three single-bit inputs (A, B, Carry-in). Outputs: Sum (S = A ⊕ B ⊕ C_in) and Carry-out (C_out = (A · B) + (B · C_in) + (A · C_in)). Implemented with two half adders and an OR gate.
  • Ripple Carry Adder: Chains full adders to add multi-bit numbers. Example: 4-bit adder for A = A₃A₂A₁A₀ and B = B₃B₂B₁B₀. Drawback: Delay due to carry propagation.
  • Subtractor:
    • Half Subtractor: Subtracts two bits (A - B). Outputs: Difference (D = A ⊕ B) and Borrow (B_out = A' · B).
    • Full Subtractor: Subtracts three bits (A - B - Borrow-in). Outputs: Difference (D = A ⊕ B ⊕ B_in) and Borrow-out (B_out = A' · B + A' · B_in + B · B_in).
    • Using Two’s Complement: Subtraction A - B is performed as A + (two’s complement of B) using an adder circuit.
  • Applications: Used in ALUs, calculators, and digital signal processing.

4.3 Decoder and Encoder

Decoders and encoders are combinational circuits used for data selection and conversion.

  • Decoder: Converts a binary input code to activate one of many outputs.
    • Example: 2-to-4 decoder has 2 inputs (A₁, A₀) and 4 outputs (D₀ to D₃). Only one output is 1 based on input. Truth table:
      A₁A₀D₀D₁D₂D₃
      001000
      010100
      100010
      110001
    • Logic: D₀ = A₁'A₀', D₁ = A₁'A₀, D₂ = A₁A₀', D₃ = A₁A₀.
  • Encoder: Converts one active input (from many) to a binary output code.
    • Example: 4-to-2 encoder has 4 inputs (D₀ to D₃) and 2 outputs (A₁, A₀). Only one input is 1. Outputs: A₁ = D₂ + D₃, A₀ = D₁ + D₃.
    • Priority Encoder: Handles multiple active inputs by prioritizing the highest-indexed input.
  • Applications: Decoders in memory addressing; encoders in keyboards and data compression.

4.4 Multiplexer and Demultiplexer

Multiplexers and demultiplexers route data by selecting inputs or distributing outputs.

  • Multiplexer (MUX): Selects one of many inputs to a single output based on select lines.
    • Example: 4-to-1 MUX with 4 inputs (D₀ to D₃), 2 select lines (S₁, S₀), and 1 output (Y). Output: Y = S₁'S₀'D₀ + S₁'S₀D₁ + S₁S₀'D₂ + S₁S₀D₃.
    • Truth Table (4-to-1 MUX):
      S₁S₀Y
      00D₀
      01D₁
      10D₂
      11D₃
  • Demultiplexer (DEMUX): Routes one input to one of many outputs based on select lines.
    • Example: 1-to-4 DEMUX with 1 input (D), 2 select lines (S₁, S₀), and 4 outputs (Y₀ to Y₃). Outputs: Y₀ = S₁'S₀'D, Y₁ = S₁'S₀D, Y₂ = S₁S₀'D, Y₃ = S₁S₀D.
  • Applications: MUX in data selection (e.g., CPU registers); DEMUX in data distribution (e.g., memory systems).

4.5 Code Conversion (8421, BCD, Gray, Excess-3, 2421)

Code conversion transforms numbers between different binary coding schemes used in digital systems.

  • 8421 (BCD): Each decimal digit is represented by a 4-bit binary code (0000 to 1001). Example: 45₁₀ = 0100 0101_BCD.
  • Gray Code: Adjacent numbers differ by one bit. Example: 3₁₀ = 010_Gray, 4₁₀ = 110_Gray. Conversion from binary: Gᵢ = Bᵢ ⊕ Bᵢ₊₁ (B₀ = G₀).
  • Excess-3: BCD code plus 3. Example: 5₁₀ = 0101_BCD = 1000_Excess-3 (0101 + 0011).
  • 2421 Code: Weighted code with weights 2, 4, 2, 1. Example: 5₁₀ = 0101_2421 (0·2 + 1·4 + 0·2 + 1·1 = 5).
  • Conversion Example: Convert BCD 0101 to Gray. BCD 0101 = 5₁₀. Binary 5 = 0101. Gray: G₃ = 0 ⊕ 1 = 1, G₂ = 1 ⊕ 0 = 1, G₁ = 0 ⊕ 1 = 1, G₀ = 1. Result: 1111_Gray.
  • Applications: BCD in calculators, Gray in error-free encoding (e.g., rotary encoders), Excess-3 in arithmetic circuits.

4.6 Programmable Logic Device (PROM)

Programmable Read-Only Memory (PROM) is a combinational logic device used to implement complex Boolean functions.

  • Definition: PROM is a memory device with a fixed AND array (decodes inputs to minterms) and a programmable OR array (connects minterms to outputs). It stores truth tables for Boolean functions.
  • Structure:
    • AND Array: Fixed, generates all possible minterms for input combinations.
    • OR Array: Programmable fuses connect minterms to outputs, defining the function.
    • Example: For F(A, B) = A'B + AB, PROM stores 1s for minterms m₁, m₃ at the output.
  • Programming: Fuses are blown to configure the OR array, making PROM one-time programmable.
  • Applications: Used in microcontrollers, lookup tables, and implementing combinational logic in embedded systems.
