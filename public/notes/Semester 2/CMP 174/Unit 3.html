Unit 3: Simplification of Boolean Functions

3.1 Minterms and Maxterms

Minterms and maxterms are standard forms of Boolean expressions used to represent any Boolean function in a canonical form.

  • Minterms: A minterm is a product (AND) of all variables in a function, where each variable appears exactly once, either complemented or uncomplemented. For n variables, there are 2^n minterms. Example: For variables A, B, minterms are A'B', A'B, AB', AB. Each corresponds to a row in the truth table where the function is 1. Notation: mᵢ, where i is the decimal equivalent of the binary input. Example: m₂ = AB' (A=1, B=0).
  • Maxterms: A maxterm is a sum (OR) of all variables, where each variable appears once, either complemented or uncomplemented. For n variables, there are 2^n maxterms. Example: For A, B, maxterms are A+B, A+B', A'+B, A'+B'. Each corresponds to a row where the function is 0. Notation: Mᵢ. Example: M₂ = A'+B (A=0, B=1).
  • Example: For F(A, B) = 1 when AB = 10, 11, the minterm form is F = A'B + AB = m₂ + m₃. The maxterm form is F = (A+B)·(A'+B') = M₀ · M₁.
  • Applications: Used to express Boolean functions systematically and simplify circuit design.

3.2 Sum of Products and Product of Sums

Sum of Products (SOP) and Product of Sums (POS) are two standard forms for representing Boolean functions.

  • Sum of Products (SOP): A Boolean function expressed as a sum (OR) of minterms. Example: F(A, B, C) = A'BC + AB'C + ABC corresponds to minterms m₃, m₅, m₇ (decimal 3, 5, 7). Derived from truth table rows where F = 1.
  • Product of Sums (POS): A Boolean function expressed as a product (AND) of maxterms. Example: F(A, B, C) = (A+B+C)·(A+B'+C)·(A'+B+C') corresponds to maxterms M₀, M₂, M₆ (rows where F = 0).
  • Conversion: Use truth table or algebraic manipulation. Example: For F = A'B + AB, SOP is direct. For POS, find rows where F = 0 (e.g., A+B, A'+B').
  • Example: Truth table for F(A, B) = 1 at AB = 01, 10. SOP: F = A'B + AB'. POS: F = (A+B)·(A'+B).
  • Applications: SOP is implemented with AND-OR circuits; POS with OR-AND circuits.

3.3 Dual and Complement of Boolean Function

The dual and complement operations transform Boolean expressions while preserving or altering their logic.

  • Dual: Replace AND with OR, OR with AND, 0 with 1, and 1 with 0, while keeping variables unchanged. Example: Dual of F = A · B + C is Fᵈ = (A + B) · C. Note: The dual does not necessarily yield the same function but is useful in circuit design.
  • Complement: The complement of a function F, denoted F', gives the opposite output (0 for 1, 1 for 0). Use De Morgan’s theorems or truth table. Example: For F = A · B, F' = (A · B)' = A' + B'. For F = A + B, F' = A' · B'.
  • Example: For F = A'B + AB', complement is F' = (A'B + AB')' = (A'B)' · (AB')' = (A + B') · (A' + B). Dual is Fᵈ = (A' + B) · (A + B').
  • Applications: Complements are used in circuit minimization; duals help convert between SOP and POS forms.

3.4 K-map & Don't Care Conditions

Karnaugh maps (K-maps) are graphical tools for simplifying Boolean expressions, and don’t care conditions help further optimize circuits.

  • K-map: A grid where each cell represents a minterm, used to simplify SOP or POS expressions by grouping 1s (or 0s). For n variables, the K-map has 2^n cells. Example: For 3 variables A, B, C, a 4x2 K-map is used.
    • Steps: Plot 1s for minterms (SOP) or 0s for maxterms (POS). Group adjacent 1s (or 0s) in powers of 2 (1, 2, 4, 8). Each group gives a simplified term.
    • Example: For F(A, B, C) = Σm(0, 1, 4, 5), plot 1s in K-map, group A'B' (m₀, m₁) and AB' (m₄, m₅). Result: F = A'B' + AB'.
  • Don't Care Conditions: Inputs where the output is unspecified (denoted X) can be treated as 0 or 1 to simplify expressions. Example: For F(A, B, C) = Σm(0, 1) + d(4, 5), include don’t cares in groups to get F = A' (grouping m₀, m₁, d₄, d₅).
  • Applications: K-maps minimize logic gates in circuit design; don’t cares optimize incomplete specifications (e.g., unused input combinations).

3.5 Two-Level and Multilevel Gate Implementations

Boolean functions can be implemented using two-level or multilevel gate circuits, balancing speed, complexity, and cost.

  • Two-Level Gate Implementation:
    • SOP (AND-OR): First level: AND gates for each minterm; second level: OR gate to sum them. Example: F = A'B + AB' uses two AND gates (for A'B, AB') and one OR gate.
    • POS (OR-AND): First level: OR gates for each maxterm; second level: AND gate to combine them. Example: F = (A+B)·(A'+B) uses two OR gates and one AND gate.
    • Advantage: Fast (only two gate delays). Drawback: May require many gates for complex functions.
  • Multilevel Gate Implementation: Uses multiple levels of gates (AND, OR, NOT, etc.) to factor expressions, reducing gate count. Example: F = A'B + AB'C can be factored as F = A'(B + BC) = A'(B + C), implemented with fewer gates but more levels.
  • Example: For F = A'BC + A'BC' + ABC, SOP needs three AND gates and one OR gate. Factored form F = A'B(C + C') + ABC = A'B + ABC = B(A' + AC) uses fewer gates with three levels.
  • Applications: Two-level for high-speed circuits (e.g., critical paths in CPUs); multilevel for cost-effective designs (e.g., ASICs).
