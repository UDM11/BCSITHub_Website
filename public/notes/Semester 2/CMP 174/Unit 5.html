Unit 5: Sequential Logic

5.1 Types of Sequential Logic Circuit

Sequential logic circuits have outputs that depend on both current inputs and previous states (memory), unlike combinational circuits, which depend only on current inputs.

  • Definition: Sequential circuits incorporate memory elements, enabling them to store and process information over time. Example: Counters, registers.
  • Types:
    • Synchronous Sequential Circuits: Operations are synchronized with a clock signal. Example: Flip-flops triggered by clock pulses.
    • Asynchronous Sequential Circuits: Operations depend on input changes without a clock. Example: Circuits using unclocked latches.
  • Components: Include flip-flops, latches, and combinational logic for state transitions.
  • Applications: Used in memory units, counters, shift registers, and state machines in processors and controllers.

5.2 Triggering of Flip-Flops

Triggering determines how and when a flip-flop changes its state based on input signals and clock events.

  • Level Triggering: Flip-flop responds to input levels (high or low). Example: Latches change state as long as the enable signal is active. Drawback: Sensitive to noise during the active level.
  • Edge Triggering: Flip-flop responds to the rising (positive) or falling (negative) edge of a clock signal. Example: Edge-triggered flip-flops update only at clock transitions, ensuring stability.
  • Types of Edge Triggering:
    • Positive Edge: State changes when clock transitions from 0 to 1.
    • Negative Edge: State changes when clock transitions from 1 to 0.
  • Example: A D flip-flop with positive edge triggering updates its output to match the D input only at the rising clock edge.
  • Applications: Edge triggering is used in synchronous circuits to ensure predictable timing in processors and memory systems.

5.3 Flip-Flops (RS, JK, T, D, Master-Slave)

Flip-flops are basic memory elements in sequential circuits, storing one bit of information.

  • RS Flip-Flop (Set-Reset):
    • Inputs: S (Set), R (Reset). Outputs: Q, Q' (complement).
    • Operation: S=1, R=0 sets Q=1; S=0, R=1 resets Q=0; S=0, R=0 holds state; S=1, R=1 is invalid (race condition).
    • Truth Table:
      SRQ(next)
      00Q
      010
      101
      11Invalid
  • JK Flip-Flop:
    • Inputs: J, K, Clock. Resolves RS invalid state. J=1, K=0 sets Q=1; J=0, K=1 resets Q=0; J=1, K=1 toggles Q; J=0, K=0 holds state.
    • Truth Table:
      JKQ(next)
      00Q
      010
      101
      11Q'
  • T Flip-Flop (Toggle):
    • Input: T, Clock. T=1 toggles Q; T=0 holds state. Derived from JK with J=K=T.
    • Truth Table:
      TQ(next)
      0Q
      1Q'
  • D Flip-Flop (Data):
    • Input: D, Clock. Output Q follows D at clock edge. Prevents invalid states.
    • Truth Table:
      DQ(next)
      00
      11
  • Master-Slave Flip-Flop: Two flip-flops (master and slave) connected in series. Master updates on clock’s active edge; slave copies master on opposite edge, preventing race conditions. Example: Master-slave JK flip-flop ensures stable toggling.
  • Applications: Flip-flops are used in registers, counters, and finite state machines.

5.4 State Diagrams and State Tables

State diagrams and state tables describe the behavior of sequential circuits, showing how states transition based on inputs.

  • State Diagram: A graphical representation where nodes represent states (flip-flop outputs), and directed edges show transitions based on inputs, labeled with input/output. Example: A 2-bit counter (states 00, 01, 10, 11) has edges showing transitions (e.g., 00 → 01 on clock pulse).
  • State Table: A tabular representation listing current states, inputs, next states, and outputs. Example: For a JK flip-flop:
    Current QJKNext Q
    0000
    0010
    0101
    0111
    1001
    1010
    1101
    1110
  • Example: A 2-bit up-counter (states 00, 01, 10, 11). State diagram: 00 → 01 → 10 → 11 → 00 on clock pulses. State table:
    Current StateInput (Clock)Next State
    00101
    01110
    10111
    11100
  • Applications: Used to design finite state machines, model sequential circuits, and analyze system behavior in controllers and processors.
