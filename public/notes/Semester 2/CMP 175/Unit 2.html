Unit 2: Basic Java

2.1 Introduction to Java: Features of Java

Java is a high-level, object-oriented programming language developed by Sun Microsystems in 1995, now maintained by Oracle. It is widely used for its portability, robustness, and versatility.

  • Platform Independence: Java programs are compiled into bytecode, which runs on any device with a Java Virtual Machine (JVM), enabling "write once, run anywhere."
  • Object-Oriented: Supports OOP principles like encapsulation, inheritance, polymorphism, and abstraction.
  • Robust: Features like automatic memory management (garbage collection), exception handling, and type checking make Java reliable.
  • Secure: Java’s sandbox environment and bytecode verification ensure security against malicious code.
  • Multithreaded: Supports concurrent execution of multiple threads, enhancing performance for multi-tasking applications.
  • Simple: Java’s syntax is clear and derived from C/C++, making it easier to learn for programmers familiar with these languages.
  • Portable: Java’s bytecode and JVM ensure consistent behavior across different platforms.

2.2 The Java Virtual Machine (JVM), Parts of Java Program, Naming Conventions in Java

Java Virtual Machine (JVM): The JVM is an abstract computing machine that interprets or compiles Java bytecode into machine code for execution on specific hardware. It provides platform independence by acting as an intermediary between Java code and the operating system.

Parts of a Java Program:

  • Class Declaration: Every Java program contains at least one class, defined using the class keyword.
  • Main Method: The entry point of a program, defined as public static void main(String[] args).
  • Variables and Methods: Variables store data, and methods define behavior within the class.
  • Comments: Single-line (//) or multi-line (/* */) comments for documentation.
  • Packages: Used to organize classes, declared with the package keyword.
  • Imports: Import statements bring in external classes (e.g., import java.util.Scanner;).

Naming Conventions in Java:

  • Class Names: Use PascalCase (e.g., MyClass).
  • Method Names: Use camelCase (e.g., calculateTotal).
  • Variables: Use camelCase (e.g., studentName).
  • Constants: Use UPPER_SNAKE_CASE (e.g., MAX_VALUE).
  • Packages: Use lowercase with dots (e.g., com.example.project).

2.3 Variables and Constants

Variables: Named memory locations that store data, which can change during program execution. Declared with a data type, name, and optional initial value (e.g., int count = 10;).

Constants: Variables whose values cannot change once assigned, declared using the final keyword (e.g., final double PI = 3.14159;).

  • Types of Variables: Local (within a method), instance (within a class, non-static), and static (shared across all instances of a class).
  • Scope: Determines where a variable is accessible (e.g., local variables are only accessible within their block).

2.4 Data Types in Java

Java has two categories of data types:

  • Primitive Data Types:
    • byte: 8-bit integer (-128 to 127).
    • short: 16-bit integer (-32,768 to 32,767).
    • int: 32-bit integer (-2^31 to 2^31-1).
    • long: 64-bit integer (-2^63 to 2^63-1).
    • float: 32-bit floating-point.
    • double: 64-bit floating-point.
    • char: 16-bit Unicode character.
    • boolean: true or false.
  • Reference Data Types: Include objects like arrays, strings, and user-defined classes, which store references to data rather than the data itself.

2.5 Operators in Java

Operators perform operations on variables and values. Java supports:

  • Arithmetic Operators: +, -, *, /, % (modulus), ++, --.
  • Relational Operators: ==, !=, >, <, >=, <=.
  • Logical Operators: && (AND), || (OR), ! (NOT).
  • Bitwise Operators: &, |, ^, ~, <<, >>, >>>.
  • Assignment Operators: =, +=, -=, *=, /=, etc.
  • Ternary Operator: condition ? value1 : value2.

2.6 Reading and Displaying Output

Displaying Output: Use System.out.println() or System.out.print() to display text or values to the console.

Reading Input: Use the Scanner class (from java.util) to read user input.

Example:

import java.util.Scanner;
public class InputOutput {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);
        System.out.print("Enter your name: ");
        String name = scanner.nextLine();
        System.out.println("Hello, " + name + "!");
        scanner.close();
    }
}

2.7 Command Line Arguments

Command line arguments are inputs passed to a Java program when it is executed. They are stored as strings in the String[] args parameter of the main method.

Example:

public class CommandLine {
    public static void main(String[] args) {
        for (String arg : args) {
            System.out.println("Argument: " + arg);
        }
    }
}

Run as: java CommandLine arg1 arg2

2.8 Control Statements in Java

Control statements manage the flow of execution:

  • Conditional Statements:
    • if, if-else, else-if ladder: Execute code based on conditions.
    • switch: Selects one of many code blocks to execute.
  • Looping Statements:
    • for: Iterates a fixed number of times.
    • while: Repeats while a condition is true.
    • do-while: Executes at least once, then repeats while a condition is true.
  • Jump Statements:
    • break: Exits a loop or switch.
    • continue: Skips the current iteration of a loop.
    • return: Exits a method and optionally returns a value.

2.9 Array

An array is a fixed-size collection of elements of the same data type, stored in contiguous memory locations.

  • Declaration: int[] numbers; or int numbers[];
  • Initialization: numbers = new int[5]; or int[] numbers = {1, 2, 3, 4, 5};
  • Accessing Elements: Use index (e.g., numbers[0]).
  • Multidimensional Arrays: Arrays of arrays (e.g., int[][] matrix = new int[3][3];).

Example:

public class ArrayExample {
    public static void main(String[] args) {
        int[] numbers = {1, 2, 3, 4, 5};
        for (int num : numbers) {
            System.out.println(num);
        }
    }
}
