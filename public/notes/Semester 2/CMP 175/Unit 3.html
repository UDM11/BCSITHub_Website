Unit 3: Object-Oriented Programming

3.1 Classes and Objects: Object Creation, Initializing the Instance Variables

A class in Java is a blueprint for creating objects, defining their properties (instance variables) and behaviors (methods). An object is an instance of a class, representing a specific entity with its own state and behavior.

Object Creation: Objects are created using the new keyword, which allocates memory and invokes a constructor.

Initializing Instance Variables: Instance variables (non-static fields) can be initialized directly, through constructors, or via setter methods.

Example:

public class Student {
    String name; // Instance variable
    int age;

    // Constructor for initialization
    public Student(String name, int age) {
        this.name = name;
        this.age = age;
    }

    public void display() {
        System.out.println("Name: " + name + ", Age: " + age);
    }

    public static void main(String[] args) {
        // Object creation
        Student s1 = new Student("Alice", 20);
        s1.display(); // Output: Name: Alice, Age: 20
    }
}

3.2 Access Specifiers and Encapsulation

Access Specifiers: Control the visibility and accessibility of class members (fields, methods, constructors).

  • public: Accessible from everywhere.
  • protected: Accessible within the same package and in subclasses (even if in different packages).
  • default: (No specifier) Accessible only within the same package.
  • private: Accessible only within the same class.

Encapsulation: The process of bundling data (instance variables) and methods that operate on the data into a single unit (class), while restricting direct access to some components. Achieved by making fields private and providing public getter/setter methods.

Example:

public class BankAccount {
    private double balance; // Private field

    // Public getter
    public double getBalance() {
        return balance;
    }

    // Public setter
    public void setBalance(double balance) {
        if (balance >= 0) {
            this.balance = balance;
        }
    }
}

3.3 Constructors and Garbage Collection

Constructors: Special methods used to initialize objects. They have the same name as the class and no return type.

  • Default Constructor: Provided automatically if no constructors are defined, initializes fields to default values (e.g., 0, null).
  • Parameterized Constructor: Accepts parameters to initialize fields.
  • Constructor Overloading: Multiple constructors with different parameter lists.

Garbage Collection: Java’s automatic memory management system that deallocates memory for objects no longer referenced. The JVM’s garbage collector identifies and frees unused objects, preventing memory leaks.

Example:

public class Car {
    String model;

    // Default constructor
    public Car() {
        model = "Unknown";
    }

    // Parameterized constructor
    public Car(String model) {
        this.model = model;
    }

    public static void main(String[] args) {
        Car c1 = new Car(); // Uses default constructor
        Car c2 = new Car("Toyota"); // Uses parameterized constructor
        c1 = null; // c1 eligible for garbage collection
    }
}

3.4 "this" Pointer

The this keyword refers to the current object in a method or constructor. It is used to:

  • Distinguish instance variables from local variables or parameters with the same name.
  • Invoke another constructor of the same class (this()).
  • Pass the current object as a parameter to methods.

Example:

public class Employee {
    String name;
    int id;

    public Employee(String name, int id) {
        this.name = name; // Differentiates instance variable from parameter
        this.id = id;
    }

    public void display() {
        System.out.println("Name: " + this.name + ", ID: " + this.id); // Refers to current object
    }

    public static void main(String[] args) {
        Employee e = new Employee("Bob", 101);
        e.display(); // Output: Name: Bob, ID: 101
    }
}

3.5 Static Members

Static Members: Belong to the class rather than any specific object. Declared with the static keyword. They are shared across all instances of the class.

  • Static Variables: Single copy shared by all objects (e.g., a counter for the number of objects created).
  • Static Methods: Can be called without creating an object, often used for utility functions.
  • Static Blocks: Used to initialize static variables, executed when the class is loaded.

Example:

public class Counter {
    static int count = 0; // Static variable

    // Constructor increments count
    public Counter() {
        count++;
    }

    // Static method to access count
    public static int getCount() {
        return count;
    }

    public static void main(String[] args) {
        Counter c1 = new Counter();
        Counter c2 = new Counter();
        System.out.println("Total objects: " + Counter.getCount()); // Output: Total objects: 2
    }
}
