Unit 5: Exception Handling

5.1 Introduction to Exception

An exception in Java is an event that disrupts the normal flow of a program’s execution. It occurs during runtime due to unexpected conditions, such as invalid input, file not found, or division by zero. Exception handling allows a program to detect and respond to these issues gracefully, ensuring robustness and preventing crashes.

Exceptions are objects derived from the Throwable class, with two main subclasses: Error and Exception.

5.2 Errors in Java Program

Errors represent serious problems that a Java program cannot reasonably recover from. They are typically caused by issues outside the program’s control, such as hardware failures or JVM issues.

  • Types of Errors:
    • OutOfMemoryError: Occurs when the JVM runs out of memory.
    • StackOverflowError: Caused by excessive recursion leading to stack overflow.
    • VirtualMachineError: Indicates JVM issues, such as corrupted internal data.
  • Errors are not typically caught or handled, as they indicate critical issues that require external intervention.

Example:

public class ErrorExample {
    public static void recursiveMethod() {
        recursiveMethod(); // Causes StackOverflowError
    }

    public static void main(String[] args) {
        recursiveMethod();
    }
}

5.3 Types of Exceptions

Exceptions in Java are categorized into two types:

  • Checked Exceptions: Checked at compile-time, these are exceptions that the programmer must handle explicitly using try-catch or declare using throws. Examples include IOException and SQLException.
  • Unchecked Exceptions: Occur at runtime and are not checked at compile-time. These are subclasses of RuntimeException, such as NullPointerException and ArithmeticException.

Example:

public class ExceptionTypes {
    public static void main(String[] args) {
        // Unchecked Exception
        int[] arr = null;
        System.out.println(arr[0]); // NullPointerException

        // Checked Exception (requires handling)
        // FileInputStream file = new FileInputStream("file.txt"); // FileNotFoundException
    }
}

5.4 Exception Handling Mechanism

Java provides a robust mechanism to handle exceptions using the following keywords:

  • try: Encloses code that might throw an exception.
  • catch: Handles the exception thrown in the try block.
  • finally: Executes code regardless of whether an exception occurs, often used for cleanup.
  • throw: Explicitly throws an exception.
  • throws: Declares exceptions that a method might throw.

Example:

public class ExceptionHandling {
    public static void main(String[] args) {
        try {
            int result = 10 / 0; // ArithmeticException
        } catch (ArithmeticException e) {
            System.out.println("Error: " + e.getMessage());
        } finally {
            System.out.println("Cleanup in finally block.");
        }
    }
}

5.5 Built-in Exceptions in Java

Java provides a variety of built-in exceptions under the java.lang package and others. Common examples include:

  • ArithmeticException: Thrown for arithmetic errors, like division by zero.
  • NullPointerException: Thrown when accessing a null object’s methods or fields.
  • ArrayIndexOutOfBoundsException: Thrown when accessing an invalid array index.
  • IOException: Thrown for I/O-related errors, such as file not found.
  • ClassNotFoundException: Thrown when a class is not found during reflection.

Example:

public class BuiltInExceptions {
    public static void main(String[] args) {
        try {
            int[] arr = {1, 2, 3};
            System.out.println(arr[5]); // ArrayIndexOutOfBoundsException
        } catch (ArrayIndexOutOfBoundsException e) {
            System.out.println("Error: " + e.getMessage());
        }
    }
}

5.6 User-Defined Exception in Java

User-defined exceptions are custom exceptions created by extending the Exception class (for checked exceptions) or RuntimeException (for unchecked exceptions). They allow developers to define specific error conditions for their application.

Example:

public class InsufficientBalanceException extends Exception {
    public InsufficientBalanceException(String message) {
        super(message);
    }
}

public class BankAccount {
    private double balance;

    public void withdraw(double amount) throws InsufficientBalanceException {
        if (amount > balance) {
            throw new InsufficientBalanceException("Insufficient balance for withdrawal.");
        }
        balance -= amount;
    }

    public static void main(String[] args) {
        BankAccount account = new BankAccount();
        account.balance = 100;
        try {
            account.withdraw(150);
        } catch (InsufficientBalanceException e) {
            System.out.println("Error: " + e.getMessage());
        }
    }
}
