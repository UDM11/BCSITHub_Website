Unit 6: Internet and Internet Services

6.1 Introduction

The Internet is a global network of interconnected computers that communicate using standardized protocols to share information and resources. It enables a wide range of services, from communication to commerce, transforming how individuals and organizations interact.

  • Purpose: Facilitates data exchange, communication, and access to resources worldwide.
  • Components: Includes networks, servers, routers, and client devices connected via protocols.
  • Examples: Services like email, web browsing, and online streaming rely on the Internet.

6.2 History of Internet

The Internet evolved from early research networks to a global communication infrastructure over several decades.

  • 1960s: ARPANET, funded by the U.S. Department of Defense, laid the foundation for packet-switching networks.
  • 1980s: TCP/IP protocols were standardized, and the term "Internet" emerged.
  • 1990s: The World Wide Web, introduced by Tim Berners-Lee, made the Internet accessible to the public.

6.3 Working Mechanism of Internet

The Internet operates by transmitting data packets across interconnected networks using standardized protocols.

  • Packet Switching: Data is divided into packets, routed independently, and reassembled at the destination.
  • Routing: Routers direct packets through the most efficient paths across networks.
  • Examples: Browsing a website involves data packets traveling from a server to a user’s device.

6.4 IP Address and Domain Name System (DNS)

IP addresses and DNS are critical for identifying and locating devices on the Internet.

  • IP Address: A unique numerical identifier for devices, e.g., IPv4 (192.168.0.1) or IPv6 (2001:db8::1).
  • Domain Name System (DNS): Translates human-readable domain names (e.g., google.com) into IP addresses.
  • Examples: Typing “www.example.com” in a browser triggers a DNS lookup to find its IP address.

6.5 Client-Server Architecture

Client-server architecture is a model where clients request services or resources from servers over a network.

  • Client: A device or software (e.g., web browser) that requests data or services.
  • Server: A device or software that provides data or services, e.g., web or email servers.
  • Examples: A browser (client) requests a webpage from a web server, which delivers the content.

6.6 Internet Protocols (HTTP, SMTP, POP, FTP, TELNET)

Internet protocols are rules governing data exchange for specific services.

  • HTTP (HyperText Transfer Protocol): Transfers web pages between servers and browsers.
  • SMTP (Simple Mail Transfer Protocol): Sends emails from clients to servers or between servers.
  • POP (Post Office Protocol): Retrieves emails from a server to a client device.
  • FTP (File Transfer Protocol): Transfers files between computers over a network.
  • TELNET: Enables remote login to another computer for command-line access.

6.7 Static and Dynamic Web Pages

Web pages are classified based on how their content is generated and displayed.

  • Static Web Pages: Fixed content delivered as stored, unchanging HTML files, e.g., informational pages.
  • Dynamic Web Pages: Content generated in real-time using scripts or databases, e.g., social media feeds.
  • Examples: A company’s “About Us” page (static) vs. an e-commerce product listing (dynamic).

6.8 Search Engines

Search engines are tools that index and retrieve information from the Internet based on user queries.

  • Function: Crawls web pages, indexes content, and ranks results based on relevance.
  • Components: Includes crawlers, databases, and ranking algorithms.
  • Examples: Google, Bing, and Yahoo Search.

6.9 E-Commerce and M-Commerce

E-commerce and M-commerce involve buying and selling goods or services over the Internet.

  • E-Commerce (Electronic Commerce): Online transactions via websites, e.g., Amazon or eBay.
  • M-Commerce (Mobile Commerce): Transactions conducted via mobile devices, e.g., payments through apps like PayPal.
  • Examples: Ordering clothes online (e-commerce) or paying for groceries via a mobile app (m-commerce).

6.10 E-Governance

E-governance refers to the use of Internet and digital technologies to deliver government services and information to citizens.

  • Purpose: Enhances transparency, efficiency, and accessibility of government processes.
  • Services: Includes online tax filing, digital identity systems, and public service portals.
  • Examples: Government websites for license renewals or online voting systems.
