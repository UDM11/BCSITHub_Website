Unit 7: Database Management System

7.1 Introduction to Database

A database is an organized collection of data, typically stored and accessed electronically from a computer system. It allows efficient storage, retrieval, and management of data, supporting multiple users and applications simultaneously.

  • Purpose: Databases enable structured data storage, efficient querying, and data integrity for various applications.
  • Components: Consists of tables (rows and columns) and a Database Management System (DBMS) to manage data.
  • Examples: MySQL, Oracle, PostgreSQL, and MongoDB are widely used databases.

7.2 Data Concepts and Characteristics

Data concepts define the fundamental principles of data organization, while characteristics highlight the qualities that make databases effective.

  • Data Concepts: Data (raw facts), information (processed data), and metadata (data about data) form the basis of databases.
  • Characteristics: Includes accuracy (error-free data), consistency (uniform data across the system), security (protection from unauthorized access), and scalability (ability to handle growing data).
  • Examples: A customer database ensures accurate records (e.g., names, addresses) and consistent formats across entries.

7.3 Database vs. File System

Databases and file systems both store data but differ in structure, functionality, and efficiency.

  • Data Organization: File systems use files and folders with no inherent structure for querying, while databases use structured tables with relationships.
  • Data Redundancy: File systems often duplicate data, leading to inconsistency; databases reduce redundancy via normalization.
  • Access Control: Databases provide advanced security and user access controls, unlike basic file-level permissions in file systems.
  • Examples: Storing records in Excel files (file system) vs. a MySQL database for efficient querying and management.

7.4 Data Models

Data models define the structure and organization of data within a database, guiding how data is stored and accessed.

  • Hierarchical Model: Organizes data in a tree-like structure with parent-child relationships, suitable for structured data like file systems.
  • Network Model: Allows multiple parent-child relationships, forming a graph structure, offering flexibility but complex management.
  • Relational Model: Uses tables with rows and columns, linked by keys, widely used for its simplicity and efficiency.
  • NoSQL Models: Includes key-value, document, or graph models, designed for unstructured data and scalability.

7.5 Database Management System

A Database Management System (DBMS) is software that facilitates the creation, management, and manipulation of databases, ensuring efficient data handling.

  • Functions: Supports data storage, retrieval, updating, and security, while enabling concurrent access by multiple users.
  • Features: Provides query processing, backup/recovery, and data integrity enforcement.
  • Examples: Oracle, MySQL, Microsoft SQL Server, and MongoDB are popular DBMSs.

7.6 RDBMS

A Relational Database Management System (RDBMS) is a DBMS that manages data organized in tables with relationships defined by keys, using Structured Query Language (SQL).

  • Key Features: Supports SQL, enforces data integrity with constraints (e.g., primary keys, foreign keys), and ensures ACID properties (Atomicity, Consistency, Isolation, Durability).
  • Structure: Data is stored in tables, with relationships established through keys for efficient querying.
  • Examples: MySQL, PostgreSQL, and Oracle Database are widely used RDBMSs.

7.7 Database Applications

Database applications leverage databases to manage data for various industries and purposes, enhancing efficiency and decision-making.

  • Business: Used in inventory management, customer relationship systems (CRM), and financial systems.
  • Healthcare: Manages patient records, medical histories, and billing information.
  • Education: Supports student information systems, online learning platforms, and research databases.
  • E-Commerce: Powers product catalogs, order processing, and customer data management.
