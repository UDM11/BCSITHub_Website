Unit 4: Operating System

4.4 Introduction

An operating system (OS) is a software that acts as an intermediary between computer hardware and users. It manages hardware resources, provides services for software applications, and enables users to interact with the computer system efficiently.

  • Purpose: The OS ensures smooth operation of the computer by managing resources and providing a user-friendly interface.
  • Components: Includes kernel, device drivers, and user interface components like graphical or command-line interfaces.
  • Examples: Popular operating systems include Windows, Linux, macOS, and Android.

4.5 Objective of Operating System

The primary objectives of an operating system are to optimize resource utilization, enhance user experience, and ensure system reliability.

  • Resource Management: Efficiently allocates CPU, memory, and storage to maximize performance.
  • User Convenience: Provides an intuitive interface for users to interact with the system.
  • System Efficiency: Ensures reliable execution of programs and minimizes system downtime.

4.6 Types of Operating System

Operating systems are classified based on their design, functionality, and intended use.

  • Batch OS: Processes jobs in batches without user interaction, used in early mainframes.
  • Multitasking OS: Allows multiple tasks to run simultaneously, e.g., Windows, Linux.
  • Real-Time OS: Processes data with strict time constraints, used in embedded systems like medical devices.
  • Distributed OS: Manages multiple computers working together, appearing as a single system.
  • Mobile OS: Designed for mobile devices, e.g., Android, iOS.

4.7 Functions of Operating System

The operating system performs essential functions to manage hardware and software resources effectively.

  • Process Management: Handles creation, scheduling, and termination of processes.
  • Memory Management: Allocates and deallocates memory for processes.
  • File Management: Organizes, stores, and retrieves files on storage devices.
  • Device Management: Controls hardware devices through drivers.
  • User Interface: Provides interfaces like GUI or CLI for user interaction.

4.8 Process Management

Process management involves handling the execution of processes (running programs) in the system.

  • Process States: Manages states like ready, running, waiting, and terminated.
  • Scheduling: Allocates CPU time to processes using algorithms like Round-Robin or Priority Scheduling.
  • Examples: Task Manager in Windows or ‘top’ command in Linux monitors active processes.

4.9 Memory Management

Memory management ensures efficient use of a computer’s memory resources for running processes.

  • Allocation: Assigns memory space to processes and deallocates it when no longer needed.
  • Virtual Memory: Uses disk space as an extension of RAM to run larger programs.
  • Examples: Paging and segmentation techniques in Windows and Linux manage memory effectively.

4.10 File Management

File management handles the organization, storage, and retrieval of data in files on storage devices.

  • File Operations: Supports creation, deletion, reading, and writing of files.
  • File Systems: Organizes files using systems like NTFS, FAT32, or ext4.
  • Examples: File Explorer in Windows or ‘ls’ command in Linux for file navigation.

4.11 Device Management

Device management controls and coordinates hardware devices connected to the computer.

  • Device Drivers: Software that enables communication between the OS and hardware devices.
  • Resource Allocation: Assigns devices like printers or USB drives to processes as needed.
  • Examples: Device Manager in Windows or ‘dmesg’ in Linux monitors hardware status.

4.12 Protection and Security

Protection and security mechanisms safeguard the system and data from unauthorized access and threats.

  • Protection: Controls access to resources, ensuring processes operate within their permissions.
  • Security: Implements authentication (e.g., passwords), encryption, and firewalls to prevent attacks.
  • Examples: User account controls in Windows or SELinux in Linux enforce security policies.

4.13 User Interface

The user interface allows users to interact with the operating system and execute tasks.

  • Graphical User Interface (GUI): Provides visual elements like windows, icons, and menus, e.g., Windows Desktop or macOS.
  • Command-Line Interface (CLI): Enables interaction through text commands, e.g., Windows PowerShell or Linux Terminal.
  • Examples: Start Menu in Windows or ‘bash’ shell in Linux for user interaction.
