Unit 2: Computer Hardware

2.1 Introduction

Computer hardware refers to the physical components of a computer system that can be seen and touched. These components work together to process data, execute instructions, and produce output. Hardware forms the foundation of any computing device, enabling it to perform tasks as directed by software.

  • Purpose: Hardware provides the physical infrastructure for data processing, storage, and interaction with users.
  • Components: Key hardware includes the CPU, memory, storage devices, input devices, and output devices.
  • Examples: Common hardware includes processors (e.g., Intel Core), RAM, hard drives, keyboards, and monitors.

2.2 Functional Block Diagram of Digital Computer System

A digital computer system is organized into functional units that work together to process data. The block diagram represents the main components: Arithmetic Logic Unit (ALU), Control Unit (CU), Memory Unit, and Input/Output Units, all interconnected to perform computing tasks.

  • Overview: The block diagram illustrates how data flows between input, processing, memory, control, and output units.
  • Data Flow: Input units accept data, the ALU and CU process it, memory stores it, and output units display or transmit results.
  • Interconnection: Components communicate via buses (data, address, and control buses) to coordinate operations.

2.2.1 ALU (Arithmetic Logic Unit)

The Arithmetic Logic Unit (ALU) is a core component of the CPU responsible for performing arithmetic and logical operations.

  • Functions: Executes arithmetic operations (e.g., addition, subtraction) and logical operations (e.g., AND, OR, NOT).
  • Data Processing: Receives data from registers or memory, processes it, and sends results back for storage or output.
  • Examples: ALUs in modern CPUs, like those in Intel or AMD processors, handle complex calculations at high speeds.

2.2.2 CU (Control Unit)

The Control Unit (CU) directs the operation of the processor, coordinating the activities of other hardware components.

  • Functions: Fetches instructions from memory, decodes them, and directs the ALU, memory, and I/O units to execute them.
  • Coordination: Manages the flow of data between the CPU, memory, and I/O devices via control signals.
  • Examples: The CU in a processor ensures that instructions are executed in the correct sequence, such as in a fetch-decode-execute cycle.

2.2.3 Memory Unit

The Memory Unit stores data and instructions temporarily or permanently for processing and retrieval.

  • Types: Primary memory (e.g., RAM, ROM) for temporary storage and secondary memory (e.g., HDD, SSD) for long-term storage.
  • Functions: Holds data and instructions being processed by the CPU and stores results for future use.
  • Examples: DDR4 RAM for fast access during processing, and SSDs for reliable, high-capacity storage.

2.2.4 Input and Output Units

Input and Output Units facilitate communication between the computer and the external environment, allowing data entry and result display.

  • Input Units: Devices like keyboards, mice, and scanners that allow users to enter data or instructions into the computer.
  • Output Units: Devices like monitors, printers, and speakers that display or produce the results of computations.
  • Examples: A touchscreen serves as both an input (touch gestures) and output (display) device, while a printer outputs hard copies of documents.
