Unit 1: Basic Concept

1.1 Elementary Logic

Elementary logic is the foundation of reasoning, used to construct and evaluate arguments systematically, particularly in mathematics and computer science.

  • Propositions: Statements that are either true or false (e.g., “2 + 2 = 4” is true; “2 + 2 = 5” is false). Non-propositional statements, like questions, are excluded.
  • Logical Arguments: Consist of premises (given statements) leading to a conclusion. An argument is valid if the conclusion follows logically from the premises.
  • Truth Values: Propositions are evaluated as true (T) or false (F), forming the basis for logical operations and proofs.
  • Applications: Used in programming (e.g., Boolean logic in conditionals), mathematics, and decision-making to ensure consistent reasoning.

Elementary logic provides tools to analyze and construct valid arguments, essential for problem-solving.

1.2 Connectives, Quantifiers

Logical connectives and quantifiers are tools to combine propositions and express relationships, forming the basis of complex logical statements.

  • Connectives: Include AND (conjunction, ∧), OR (disjunction, ∨), NOT (negation, ¬), IMPLIES (implication, →), and IF AND ONLY IF (biconditional, ↔). For example, “p ∧ q” is true only if both p and q are true.
  • Truth Tables: Used to evaluate connectives by listing all possible truth values (e.g., for p → q, the statement is false only when p is true and q is false).
  • Quantifiers: Express scope of variables. Universal quantifier (∀, “for all”) applies to every element (e.g., ∀x (x > 0)), while existential quantifier (∃, “there exists”) applies to at least one (e.g., ∃x (x² = 4)).
  • Applications: Connectives and quantifiers are used in programming logic, database queries, and mathematical proofs to define conditions and relationships.

These tools enable precise formulation of logical expressions in reasoning and computation.

1.3 Basic Laws of Logic

Basic laws of logic govern how propositions and connectives interact, ensuring consistent and valid reasoning in mathematical and computational contexts.

  • Commutative Law: Order doesn’t matter in AND or OR operations (e.g., p ∧ q = q ∧ p; p ∨ q = q ∨ p).
  • Associative Law: Grouping doesn’t affect AND or OR (e.g., (p ∧ q) ∧ r = p ∧ (q ∧ r)).
  • Distributive Law: AND and OR distribute over each other (e.g., p ∧ (q ∨ r) = (p ∧ q) ∨ (p ∧ r)).
  • De Morgan’s Laws: Negation of conjunctions and disjunctions: ¬(p ∧ q) = ¬p ∨ ¬q; ¬(p ∨ q) = ¬p ∧ ¬q. These are critical for simplifying logical expressions.

These laws ensure logical equivalence and simplify complex expressions in proofs and algorithms.

1.4 Quantifiers

Quantifiers specify the scope of variables in logical statements, crucial for precise mathematical and computational reasoning.

  • Universal Quantifier (∀): Indicates a statement is true for all elements in a set (e.g., ∀x ∈ ℝ (x² ≥ 0) means “for all real numbers x, x² is non-negative”).
  • Existential Quantifier (∃): Indicates a statement is true for at least one element (e.g., ∃x ∈ ℝ (x² = 9) means “there exists a real number x such that x² = 9”).
  • Negation of Quantifiers: Negating a universal quantifier yields an existential one, and vice versa (e.g., ¬∀x P(x) = ∃x ¬P(x)).
  • Applications: Used in formal specifications, database queries (e.g., SQL), and proofs to define conditions over sets or domains.

Quantifiers enable precise expression of generalizations and existence in logical arguments.

1.5 Techniques of Proof

Techniques of proof are methods to establish the truth of mathematical statements, widely used in logic, mathematics, and computer science.

  • Direct Proof: Assume premises are true and logically derive the conclusion (e.g., prove “if n is even, then n² is even” by showing n = 2k implies n² = 4k²).
  • Proof by Contradiction: Assume the opposite of the conclusion, derive a contradiction (e.g., to prove √2 is irrational, assume it’s rational and show it leads to a contradiction).
  • Mathematical Induction: Prove a statement for a base case and show it holds for the next case (e.g., prove Σn = n(n+1)/2 for all n ≥ 1).
  • Counterexample: Disprove a universal statement by providing a single example where it fails (e.g., “all primes are odd” is disproved by n = 2).

These techniques ensure rigorous validation of claims in theoretical and applied fields.

1.6 Sets, Types of Sets, Venn Diagram

Sets are collections of distinct elements, fundamental to mathematics and computer science, with Venn diagrams visually representing relationships.

  • Sets: Defined by elements (e.g., A = {1, 2, 3}) or properties (e.g., {x | x is even}). Notation includes ∈ for membership (e.g., 2 ∈ A).
  • Types of Sets: Include empty set (∅), finite sets (e.g., {1, 2}), infinite sets (e.g., ℕ), universal set (all elements in context), and subsets (B ⊆ A if all elements of B are in A).
  • Venn Diagrams: Use circles to represent sets and their relationships, such as intersections (shared elements) or unions (all elements). For example, two overlapping circles show A ∩ B.
  • Applications: Sets are used in databases, programming (e.g., sets in Python), and logic to organize and analyze data.

Sets and Venn diagrams provide a framework for understanding relationships and structures.

1.7 Set Operations, Laws of the Algebra of Sets

Set operations manipulate sets to form new ones, governed by algebraic laws, essential for data analysis and logical reasoning.

  • Operations: Union (A ∪ B: all elements in A or B), intersection (A ∩ B: elements in both A and B), difference (A \ B: elements in A but not B), and complement (A’: elements not in A).
  • Commutative Law: A ∪ B = B ∪ A; A ∩ B = B ∩ A.
  • Associative Law: (A ∪ B) ∪ C = A ∪ (B ∪ C); (A ∩ B) ∩ C = A ∩ (B ∩ C).
  • De Morgan’s Laws for Sets: (A ∪ B)’ = A’ ∩ B’; (A ∩ B)’ = A’ ∪ B’. These simplify set expressions and are used in programming and database queries.

Set operations and laws enable efficient manipulation and analysis of data collections.

1.8 Real Number System, Representation on the Real Line

The real number system encompasses all numbers on the number line, used extensively in mathematics and computing for measurements and calculations.

  • Real Numbers: Include integers, rational numbers (e.g., 1/2), and irrational numbers (e.g., √2, π), forming a complete, ordered system.
  • Real Line Representation: Each real number corresponds to a unique point on an infinite line, with 0 as the origin, positive numbers to the right, and negative to the left.
  • Subsets: Naturals (ℕ: 1, 2, …), integers (ℤ: …, -1, 0, 1, …), rationals (ℚ: fractions), and irrationals (non-repeating decimals).
  • Applications: Real numbers are used in algorithms, data modeling, and graphical representations in programming and engineering.

The real number system and its visual representation provide a foundation for quantitative analysis.

1.9 Properties of Real Numbers, Ordered Sets, Inequalities

Properties of real numbers and their ordering underpin mathematical operations and comparisons, critical for logical and computational tasks.

  • Properties: Commutative (a + b = b + a), associative ((a + b) + c = a + (b + c)), distributive (a(b + c) = ab + ac), and identity (a + 0 = a; a × 1 = a).
  • Ordered Sets: Real numbers are totally ordered; for any a, b, either a < b, a = b, or a > b, enabling consistent comparisons.
  • Inequalities: Express relationships like a ≤ b or a > b. Properties include transitivity (if a < b and b < c, then a < c) and addition (if a < b, then a + c < b + c).
  • Applications: Used in optimization problems, algorithm design, and data sorting to establish relationships and constraints.

These properties ensure reliable calculations and comparisons in mathematical and computational contexts.

1.10 Intervals, Absolute Value, Cartesian Product, Relation

Intervals, absolute values, Cartesian products, and relations are key concepts for defining ranges, distances, and relationships in mathematics and computer science.

  • Intervals: Subsets of real numbers, e.g., closed [a, b] (includes a, b), open (a, b) (excludes a, b). Used to define ranges in programming and analysis.
  • Absolute Value: |x| represents the distance from x to 0 (e.g., |-3| = 3). Used in error calculations and optimization algorithms.
  • Cartesian Product: For sets A and B, A × B is the set of all ordered pairs (a, b) where a ∈ A, b ∈ B. Used in databases and coordinate systems.
  • Relation: A subset of A × B, defining how elements of A relate to B (e.g., {(1, a), (2, b)}). Used in graphs, databases, and function definitions.

These concepts provide tools for modeling relationships and constraints in theoretical and applied fields.
