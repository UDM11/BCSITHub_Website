Unit 7: Functions

7.1 Introduction to Functions

Functions are self-contained blocks of code designed to perform specific tasks, promoting modularity and reusability in programs.

  • Purpose: Break complex problems into smaller, manageable units.
  • Benefits: Code reusability, easier debugging, and improved readability.
  • Example: A function to calculate the square of a number.

7.2 Types of Functions

Functions in C are classified into two types: library functions and user-defined functions.

7.2.1 Library Functions

Library functions are pre-defined in C standard libraries, accessible via header files.

  • Examples: printf() (stdio.h), sqrt() (math.h), strlen() (string.h).
  • Usage: Include the appropriate header file and call the function. 
    #include <stdio.h>
#include <math.h>
int main() {
    printf("Square root of 16: %.2f\n", sqrt(16));
    return 0;
}

7.2.2 User-Defined Functions

User-defined functions are created by programmers to perform specific tasks.

  • Advantages: Tailored to specific needs, enhances code organization.
  • Example: 
    #include <stdio.h>
int square(int n) {
    return n * n;
}
int main() {
    printf("Square of 5: %d\n", square(5));
    return 0;
}

7.3 Components of Functions

A function consists of three main components: prototype, call, and definition.

7.3.1 Function Prototype

A function prototype declares the function’s return type, name, and parameters before its use.

  • Syntax: return_type function_name(parameter_list);
  • Example: 
    int add(int, int); // Prototype
  • Purpose: Informs the compiler about the function’s signature.

7.3.2 Function Call

A function call invokes the function, passing arguments if required.

  • Syntax: function_name(arguments);
  • Example: 
    #include <stdio.h>
int add(int, int);
int main() {
    int sum = add(3, 4); // Function call
    printf("Sum: %d\n", sum);
    return 0;
}
int add(int a, int b) {
    return a + b;
}

7.3.3 Function Definition

The function definition contains the actual code to be executed.

  • Syntax: 
    return_type function_name(parameter_list) {
    // Code block
    return value; // If return_type is not void
}
  • Example: See the add() function in the above example.

7.4 Function Arguments and Return Types

Functions can accept arguments and return values to communicate with the caller.

  • Arguments:
    • Pass by Value: Copies of arguments are passed; changes do not affect original variables.
    • Pass by Reference: Uses pointers to modify original variables.
  • Return Types: void (no return), int, float, etc. 
    #include <stdio.h>
void swap(int *a, int *b) { // Pass by reference
    int temp = *a;
    *a = *b;
    *b = temp;
}
int main() {
    int x = 5, y = 10;
    swap(&x, &y);
    printf("x: %d, y: %d\n", x, y); // x: 10, y: 5
    return 0;
}

7.5 Passing Arrays to Function

Arrays are passed to functions by reference, allowing modifications to the original array.

  • Syntax: Use array name or pointer in parameter. 
    #include <stdio.h>
void printArray(int arr[], int size) {
    for (int i = 0; i < size; i++) {
        printf("%d ", arr[i]);
    }
}
int main() {
    int arr[] = {1, 2, 3, 4, 5};
    printArray(arr, 5);
    return 0;
}
  • Note: Array size is often passed as a separate parameter since arrays decay to pointers.

7.6 Passing Strings to Function

Strings (character arrays) are passed similarly to arrays, as pointers to the first character.

  • Example: 
    #include <stdio.h>
void printString(char str[]) {
    printf("String: %s\n", str);
}
int main() {
    char str[] = "Hello";
    printString(str);
    return 0;
}
  • Note: Strings must be null-terminated (\0).

7.7 Storage Class (auto, register, static, extern)

Storage classes define the scope, lifetime, and storage location of variables.

  • auto: Default for local variables, allocated on stack, destroyed when scope ends. 
    auto int x = 5; // Same as int x = 5;
  • register: Suggests storing variable in CPU register for faster access (compiler may ignore). 
    register int counter = 0;
  • static: Retains value between function calls, initialized once. 
    #include <stdio.h>
void count() {
    static int c = 0; // Retains value
    c++;
    printf("%d ", c);
}
int main() {
    count(); // 1
    count(); // 2
    return 0;
}
  • extern: Declares a variable defined in another file. 
    // file1.c
int global = 10;
// file2.c
extern int global;

7.8 Scope of Variables (Local and Global)

Scope determines where a variable is accessible in a program.

  • Local Variables: Defined within a block or function, accessible only there. 
    #include <stdio.h>
int main() {
    int x = 5; // Local to main
    printf("%d\n", x);
    return 0;
}
  • Global Variables: Defined outside functions, accessible throughout the program. 
    #include <stdio.h>
int global = 10; // Global
void func() {
    printf("%d\n", global);
}
int main() {
    func();
    return 0;
}
  • Note: Local variables take precedence over global variables with the same name.

7.9 Recursive Function

A recursive function calls itself to solve a problem by breaking it into smaller instances.

  • Components: Base case (stops recursion), recursive case (calls itself).
  • Example (Factorial): 
    #include <stdio.h>
int factorial(int n) {
    if (n == 0 || n == 1) { // Base case
        return 1;
    }
    return n * factorial(n - 1); // Recursive case
}
int main() {
    printf("Factorial of 5: %d\n", factorial(5)); // 120
    return 0;
}
  • Caution: Ensure a base case to avoid infinite recursion.
