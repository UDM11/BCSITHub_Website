Unit I: Introduction to Web Technology

1.1 Web Basics

Web technology encompasses tools, protocols, and systems used to create, manage, and access content on the Internet. It forms the foundation of how information is shared and accessed online.

1.1.1 Internet, Intranet, WWW

The Internet, Intranet, and World Wide Web (WWW) are interconnected concepts that enable global and local communication and data sharing.

  • Internet: A global network of interconnected computers that communicate using standardized protocols, enabling data exchange worldwide.
  • Intranet: A private network within an organization, using Internet technologies to share information securely among employees.
  • WWW (World Wide Web): A service on the Internet that allows users to access interlinked documents (web pages) via browsers using HTTP/HTTPS.

1.1.2 Static and Dynamic Web Page

Web pages are classified based on how their content is generated and delivered to users.

  • Static Web Page: Fixed content delivered as pre-built HTML files, unchanging unless manually edited, e.g., informational pages.
  • Dynamic Web Page: Content generated in real-time using scripts or databases, tailored to user interactions, e.g., social media feeds.
  • Examples: A company’s contact page (static) vs. an online shopping cart (dynamic).

1.1.3 Web Clients

Web clients are software or devices that request and display content from web servers.

  • Function: Sends requests to servers and renders responses, typically web pages or media.
  • Types: Includes web browsers, mobile apps, or IoT devices accessing web services.
  • Examples: Google Chrome, Firefox, or a smart TV accessing streaming services.

1.1.4 Web Servers

Web servers are hardware or software systems that store, process, and deliver web content to clients upon request.

  • Function: Hosts websites, processes client requests, and sends responses like HTML pages or files.
  • Software: Includes Apache, Nginx, or Microsoft IIS for handling HTTP requests.
  • Examples: A server hosting a website like amazon.com or a local server for an intranet.

1.2 Client-Server Architecture

Client-server architecture is a model where clients request services or resources from servers over a network, forming the basis of most web applications.

  • Overview: Clients initiate communication, and servers respond with data or services.
  • Benefits: Centralizes data management and enables scalable, distributed systems.

1.2.1 Single Tier

Single-tier architecture combines all components (client, server, and database) on a single system.

  • Characteristics: Simple, self-contained, but limited in scalability and resource sharing.
  • Use Case: Standalone applications like a desktop-based calculator program.
  • Examples: A local database application running entirely on one computer.

1.2.2 Two-Tier

Two-tier architecture separates the client (user interface) and server (data storage/processing) into two layers.

  • Characteristics: Client handles presentation and logic, while the server manages data storage and retrieval.
  • Use Case: Client applications accessing a centralized database.
  • Examples: A desktop application querying a remote database server.

1.2.3 Multi-Tier

Multi-tier architecture divides functionality into three or more layers, typically presentation, application logic, and data storage.

  • Characteristics: Enhances scalability, flexibility, and maintainability by separating concerns.
  • Use Case: Complex web applications with distributed processing.
  • Examples: E-commerce platforms with separate layers for user interface, business logic, and database.

1.3 HTTP: HTTP Request and Response

HTTP (HyperText Transfer Protocol) is a protocol for transmitting data over the web, governing client-server communication.

  • HTTP Request: Sent by the client to the server, specifying an action (e.g., GET, POST) and resource (e.g., a webpage).
  • HTTP Response: Server’s reply to the client, containing requested data or status (e.g., 200 OK, 404 Not Found).
  • Examples: A browser sending a GET request for a webpage and receiving an HTML response.

1.4 URL

A URL (Uniform Resource Locator) is a web address that specifies the location of a resource on the Internet.

  • Components: Includes protocol (e.g., https), domain (e.g., example.com), path, and optional parameters.
  • Function: Directs clients to specific resources like web pages, images, or files.
  • Examples: https://www.example.com/page.html or ftp://server.com/file.txt.

1.5 Client-Side Scripting

Client-side scripting involves executing scripts in the user’s browser to enhance web page interactivity and functionality.

  • Languages: Primarily JavaScript, with frameworks like React or Vue.js.
  • Functions: Handles user interactions, validates input, and updates content without server requests.
  • Examples: Form validation or dynamic menus on a webpage.

1.6 Server-Side Scripting

Server-side scripting involves executing scripts on the server to generate dynamic web content before sending it to the client.

  • Languages: Includes PHP, Python (Django), Ruby (Rails), or Node.js.
  • Functions: Processes data, interacts with databases, and generates customized responses.
  • Examples: Generating user-specific content like a shopping cart or retrieving database records.

1.7 Web 1.0, Web 2.0, and Web 3.0

The evolution of the web is categorized into three phases based on functionality and user interaction.

  • Web 1.0: Static, read-only web with limited interactivity, focused on delivering information (e.g., early informational websites).
  • Web 2.0: Interactive, user-driven web with social media, user-generated content, and collaboration (e.g., YouTube, Facebook).
  • Web 3.0: Decentralized, AI-driven, and semantic web with enhanced personalization and blockchain integration (e.g., decentralized apps, smart contracts).
