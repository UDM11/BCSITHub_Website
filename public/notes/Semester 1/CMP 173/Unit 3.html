Unit 3: HTML5

3.1 HTML5 APIs

HTML5 introduces a variety of APIs that enhance web application functionality, allowing developers to create interactive and dynamic experiences directly in the browser.

  • Geolocation API: Retrieves the user's geographical location with permission.
  • Canvas API: Enables 2D drawing and animations using the <canvas> element.
  • Web Storage API: Provides localStorage and sessionStorage for storing data in the browser.
  • Drag and Drop API: Supports dragging and dropping elements within or across applications.
<canvas id="myCanvas" width="200" height="100"></canvas>
<script>
  const canvas = document.getElementById('myCanvas');
  const ctx = canvas.getContext('2d');
  ctx.fillStyle = 'blue';
  ctx.fillRect(50, 50, 100, 50);
</script>

3.2 HTML5 Forms

HTML5 forms introduce new input types, attributes, and validation features to improve user interaction and data collection.

  • New Input Types: email, url, date, range, etc., for specific data entry.
  • Attributes: required, placeholder, pattern for validation and user guidance.
  • Datalist: Provides a predefined list of options for input fields.
<form>
  <label for="email">Email:</label>
  <input type="email" id="email" name="email" required placeholder="Enter your email">
  <label for="color">Favorite Color:</label>
  <input type="color" id="color" name="color">
  <input type="submit" value="Submit">
</form>

3.3 Responsive Web Design

Responsive Web Design (RWD) ensures web pages render well on various devices and screen sizes using flexible layouts, images, and CSS media queries.

  • Viewport Meta Tag: Controls layout on mobile browsers.
  • Media Queries: Apply CSS styles based on device characteristics.
  • Flexible Grids: Use relative units like percentages or vw/vh.
<meta name="viewport" content="width=device-width, initial-scale=1.0">
<style>
  .container {
    width: 100%;
    max-width: 1200px;
  }
  @media (max-width: 600px) {
    .container {
      padding: 10px;
    }
  }
</style>
<div class="container">Responsive Content</div>

3.4 HTML5

HTML5 is the latest version of HTML, offering new elements, APIs, and features to support modern web development, with improved multimedia and semantic capabilities.

  • New Elements: <article>, <section>, <audio>, <video>.
  • Improved Accessibility: Better support for screen readers and assistive technologies.
  • Backward Compatibility: Works with older browsers, though some features may not be supported.
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>HTML5 Example</title>
</head>
<body>
  <article>
    <h1>HTML5 Page</h1>
    <video controls>
      <source src="video.mp4" type="video/mp4">
    </video>
  </article>
</body>
</html>

3.5 Semantic Markup

Semantic markup in HTML5 uses elements that clearly describe their meaning, improving accessibility, SEO, and code readability.

  • Semantic Elements: <header>, <footer>, <main>, <aside>.
  • Benefits: Enhances search engine understanding and assistive technology support.
  • ARIA Roles: Augment semantics for better accessibility.
<header>
  <h1>Website Title</h1>
</header>
<main>
  <article>
    <h2>Main Content</h2>
    <p>This is the primary content.</p>
  </article>
</main>
<footer>
  <p>Copyright 2025</p>
</footer>

3.6 Best Practices and Optimization

Adopting best practices in HTML5 development ensures efficient, maintainable, and accessible web pages.

  • Minimize Code: Remove unnecessary tags and use concise attributes.
  • Validate HTML: Use tools like W3C Validator to ensure standards compliance.
  • Optimize Images: Use compressed formats and appropriate sizes.
  • Accessibility: Include alt text, ARIA roles, and keyboard navigation support.
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Optimized Page</title>
</head>
<body>
  <main aria-label="Main content">
    <img src="image.jpg" alt="Description" width="300">
    <p>Accessible and optimized content.</p>
  </main>
</body>
</html>
