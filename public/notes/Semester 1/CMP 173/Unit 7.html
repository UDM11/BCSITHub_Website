Unit 7: Advanced Topics on JavaScript

7.1 Scope and Closures

Scope determines the accessibility of variables, while closures allow functions to retain access to their outer scope's variables even after the outer function has finished executing.

  • Scope: Global, function, and block scope (using let and const).
  • Closures: A function bundled with its lexical environment, retaining access to variables from its parent scope.
  • Use Cases: Data privacy, event handlers, and callbacks.
function outer() {
  let count = 0;
  return function inner() {
    return count++;
  };
}
const counter = outer();
console.log(counter()); // 0
console.log(counter()); // 1

7.2 Error Handling and Debugging

Error handling manages runtime errors, while debugging helps identify and fix issues in code.

  • try/catch: Catches and handles errors gracefully.
  • throw: Creates custom errors.
  • Debugging: Use browser tools like console, breakpoints, and debugger.
try {
  let result = undefinedVar;
} catch (error) {
  console.error('Error:', error.message);
}
throw new Error('Custom error');
console.log('This will not run');

7.3 DOM Manipulation

The Document Object Model (DOM) allows JavaScript to interact with HTML elements, enabling dynamic content updates.

  • Selecting Elements: getElementById, querySelector.
  • Modifying Elements: Change content, styles, or attributes.
  • Event Listeners: Respond to user actions like clicks or input.
const btn = document.querySelector('#myButton');
btn.addEventListener('click', () => {
  document.querySelector('p').textContent = 'Updated!';
  btn.style.backgroundColor = 'blue';
});

7.4 Asynchronous JavaScript

Asynchronous JavaScript handles operations that take time, such as fetching data, without blocking execution.

  • Callbacks: Functions passed as arguments to handle async results.
  • Promises: Objects representing eventual completion or failure.
  • async/await: Syntactic sugar for working with Promises.
async function fetchData() {
  try {
    const response = await fetch('https://api.example.com/data');
    const data = await response.json();
    console.log(data);
  } catch (error) {
    console.error('Error:', error);
  }
}
fetchData();

7.5 JSON and AJAX

JSON (JavaScript Object Notation) is a lightweight data format, and AJAX (Asynchronous JavaScript and XML) enables asynchronous data fetching.

  • JSON: Used for data exchange, parsed with JSON.parse and JSON.stringify.
  • AJAX: Uses XMLHttpRequest or fetch to load data without refreshing.
const data = JSON.stringify({ name: 'John', age: 30 });
console.log(JSON.parse(data)); // { name: 'John', age: 30 }
fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => console.error(error));

7.6 ES6 and Modern JavaScript

ES6 (ECMAScript 2015) introduced modern JavaScript features to improve code readability and functionality.

  • let/const: Block-scoped variable declarations.
  • Arrow Functions: Concise function syntax.
  • Destructuring: Extract values from arrays/objects.
  • Modules: Import/export for modular code.
const [a, b] = [1, 2]; // Destructuring
const add = (x, y) => x + y; // Arrow function
import { myFunction } from './module.js';
export const myVar = 42;

7.7 JavaScript Libraries

JavaScript libraries provide pre-written code to simplify development and enhance functionality.

  • jQuery: Simplifies DOM manipulation and AJAX.
  • React: Component-based library for building user interfaces.
  • Lodash: Utility library for common tasks like array manipulation.
  • Inclusion: Use via CDN or package managers like npm.
<script src="https://code.jquery.com/jquery-3.6.0.min.js"></script>
<script>
  $(document).ready(function() {
    $('p').hide();
  });
</script>
