Unit 7: Advanced PHP Concepts

7.1 Object-oriented Programming (OOP) in PHP

OOP in PHP allows organizing code into reusable, modular units called **classes** and **objects**. PHP has supported OOP since PHP 5 (2004) and continues to improve it significantly in PHP 7.x and 8.x (typed properties, constructor property promotion, attributes, enums, readonly properties, etc.).

Core OOP Principles Supported in PHP

  • Encapsulation (data hiding)
  • Inheritance (extends)
  • Polymorphism (method overriding, interfaces)
  • Abstraction (abstract classes, interfaces)
PHP is **not** a purely object-oriented language — it supports both procedural and OOP styles. Modern frameworks (Laravel, Symfony) are almost fully OOP-based.

7.2 Creating and Using Classes and Objects

<?php
// Class definition
class Student {
    // Properties (attributes)
    public $name;
    public $roll;
    public $gpa;

    // Constructor (called automatically when object is created)
    public function __construct($name, $roll, $gpa) {
        $this->name = $name;
        $this->roll = $roll;
        $this->gpa  = $gpa;
    }

    // Method (behavior)
    public function getDetails() {
        return "Name: {$this->name}, Roll: {$this->roll}, GPA: {$this->gpa}";
    }
}

// Creating objects (instances)
$s1 = new Student("Ram Sharma", "075BCT001", 3.85);
$s2 = new Student("Sita Gurung", "075BCT015", 3.92);

// Accessing properties & methods
echo $s1->name . "
";               // Ram Sharma
echo $s1->getDetails() . "
";       // Name: Ram Sharma, Roll: 075BCT001, GPA: 3.85
echo $s2->getDetails();                // Name: Sita Gurung, Roll: 075BCT015, GPA: 3.92
?>
PHP 8+ Constructor Property Promotion (very clean): 
class Student {
    public function __construct(
        public string $name,
        public string $roll,
        public float $gpa
    ) {}
}
Automatically creates and assigns public properties.

class Person {
    public $name;           // accessible from anywhere
    protected $age;         // accessible in class & subclasses
    private $ssn;           // only inside Person class

    public function setAge($age) {
        if ($age >= 0) {
            $this->age = $age;
        }
    }

    public function getAge() {
        return $this->age;
    }
}

class Student extends Person {
    public function display() {
        echo $this->name . ", Age: " . $this->age;     // OK (protected)
        // echo $this->ssn;                            // Error! private
    }
}

7.4 Inheritance, Encapsulation, and Polymorphism

Inheritance (extends)

class Person {
    public $name;
    protected $age;

    public function __construct($name, $age) {
        $this->name = $name;
        $this->age  = $age;
    }

    public function introduce() {
        return "Hi, I'm {$this->name}, age {$this->age}";
    }
}

class Student extends Person {
    public $roll;
    public $gpa;

    public function __construct($name, $age, $roll, $gpa) {
        parent::__construct($name, $age);     // call parent constructor
        $this->roll = $roll;
        $this->gpa  = $gpa;
    }

    // Method overriding
    public function introduce() {
        return parent::introduce() . ", Roll: {$this->roll}, GPA: {$this->gpa}";
    }
}

$s = new Student("Hari", 21, "075BCT010", 3.65);
echo $s->introduce();

Encapsulation

  • Bundling data (properties) and methods that operate on data
  • Hide internal details using private/protected
  • Expose only necessary interface (public methods)

Polymorphism

  • Same method name behaves differently in different classes
  • Achieved via method overriding
  • Also via interfaces (implements)
interface Printable {
    public function printInfo();
}

class Student implements Printable {
    public function printInfo() {
        echo "Student info...";
    }
}

class Teacher implements Printable {
    public function printInfo() {
        echo "Teacher info...";
    }
}

// Polymorphism in action
function displayInfo(Printable $obj) {
    $obj->printInfo();
}

7.5 Static Properties and Methods

class Counter {
    public static $count = 0;

    public static function increment() {
        self::$count++;
    }

    public static function getCount() {
        return self::$count;
    }
}

// No need to create object
Counter::increment();
Counter::increment();
echo Counter::getCount();      // 2

// Static properties are shared across all instances
$c1 = new Counter();
$c2 = new Counter();
echo Counter::$count;          // still 2
Common uses of static:
• Counters, configuration values
• Factory methods
• Utility/helper methods (e.g. Math::sqrt())
• Singleton pattern (though dependency injection is preferred now)

7.6 Magic Methods

Magic methods are special methods that are called automatically in certain situations.

__clone()Object cloned (clone $obj)Deep copy control
class Person {
    private $data = [];

    public function __set($key, $value) {
        $this->data[$key] = $value;
    }

    public function __get($key) {
        return $this->data[$key] ?? null;
    }

    public function __toString() {
        return "Person: " . ($this->data['name'] ?? 'Unknown');
    }
}

$p = new Person();
$p->name = "Sita";
$p->age  = 20;

echo $p;           // Person: Sita
echo $p->age;      // 20

7.7 Practical Applications of OOP in PHP Development

  • Database Abstraction: Create a Database class with connect, query, prepare methods
  • User Management: User / Admin / Student classes with inheritance
  • Form Validation: Validator class with rules (required, email, min_length...)
  • Authentication System: Auth class with login, logout, isLoggedIn methods
  • Model-View-Controller (MVC): Models (data), Controllers (logic), Views (HTML)
  • Frameworks: Laravel (Eloquent ORM), Symfony (Entities), CodeIgniter 4 (Models)
  • API Development: Controller classes with methods mapped to endpoints
  • File Upload Handler: Upload class with validation, resize, storage logic
Modern PHP (2025) OOP trend:
• Use typed properties & return types
• Constructor property promotion
• Readonly properties & classes
• Enums (PHP 8.1+)
• Attributes (annotations)
• Dependency Injection → avoid new Class() everywhere
