Unit 6: Working with Database

6.1 Introduction to MySQL Database Management System

MySQL is the world's most popular open-source relational database management system (RDBMS). It is widely used in web applications, especially with PHP (the classic LAMP stack: Linux, Apache, MySQL, PHP).

Key Features of MySQL (2025 perspective)

  • Supports standard SQL (with many extensions)
  • High performance for read-heavy workloads
  • ACID compliance (InnoDB storage engine – default since MySQL 5.5)
  • Multiple storage engines: InnoDB, MyISAM, Memory, CSV, Archive...
  • Replication (master-slave, group replication)
  • JSON data type support (very powerful since 5.7)
  • Common Table Expressions (CTE) and Window functions (8.0+)
  • Strong community + enterprise edition (Oracle MySQL Enterprise)
  • Forks: MariaDB (community-driven, drop-in replacement)
Popular in Nepal & education: Almost all college projects, WordPress sites, small to medium e-commerce, school/college management systems use MySQL + PHP.

Basic MySQL Terminology

  • Database → collection of tables
  • Table → collection of rows (records) and columns (fields)
  • Row / Record → one entry
  • Column / Field → attribute
  • Primary Key → unique identifier
  • Foreign Key → links tables

6.2 Connecting PHP with MySQL Database

There are three main ways to connect PHP to MySQL (2025 recommendation: use PDO or MySQLi with prepared statements).

Modern Recommended Way: PDO (PHP Data Objects)

<?php
$host     = 'localhost';
$dbname   = 'college_portal';
$username = 'root';
$password = '';           // default XAMPP/WAMP → empty

try {
    $pdo = new PDO("mysql:host=$host;dbname=$dbname;charset=utf8mb4", 
                   $username, $password);
    $pdo->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);
    // echo "Connected successfully!";
} catch (PDOException $e) {
    die("Connection failed: " . $e->getMessage());
}
?>

Alternative: MySQLi (Procedural or Object-oriented)

<?php
// Object-oriented style
$mysqli = new mysqli("localhost", "root", "", "college_portal");

if ($mysqli->connect_error) {
    die("Connection failed: " . $mysqli->connect_error);
}

// Procedural style
$conn = mysqli_connect("localhost", "root", "", "college_portal")
    or die("Connection failed: " . mysqli_connect_error());
?>
PDO vs MySQLi (2025 advice):
• PDO → database-agnostic (can switch to PostgreSQL easily), named parameters, better exception handling
• MySQLi → slightly faster for MySQL-only, supports multiple statements
→ Use **PDO** for most college projects

6.3 Performing CRUD Operations (Create, Read, Update, Delete)

Using PDO – Full CRUD Example

<?php
// db.php - connection file
require_once 'db.php';

// CREATE
if (isset($_POST['add_student'])) {
    $name = $_POST['name'];
    $roll = $_POST['roll'];
    $gpa  = $_POST['gpa'];

    $stmt = $pdo->prepare("INSERT INTO students (name, roll, gpa) VALUES (?, ?, ?)");
    $stmt->execute([$name, $roll, $gpa]);
    echo "Student added!";
}

// READ (all)
$stmt = $pdo->query("SELECT * FROM students ORDER BY id DESC");
$students = $stmt->fetchAll(PDO::FETCH_ASSOC);

// READ (one)
$id = 5;
$stmt = $pdo->prepare("SELECT * FROM students WHERE id = ?");
$stmt->execute([$id]);
$student = $stmt->fetch(PDO::FETCH_ASSOC);

// UPDATE
if (isset($_POST['update'])) {
    $id   = $_POST['id'];
    $name = $_POST['name'];
    $gpa  = $_POST['gpa'];

    $stmt = $pdo->prepare("UPDATE students SET name = ?, gpa = ? WHERE id = ?");
    $stmt->execute([$name, $gpa, $id]);
}

// DELETE
if (isset($_GET['delete'])) {
    $id = $_GET['delete'];
    $stmt = $pdo->prepare("DELETE FROM students WHERE id = ?");
    $stmt->execute([$id]);
    header("Location: index.php");
}
?>
Always use prepared statements with placeholders (? or :name) to prevent SQL Injection.

6.4 Executing SQL Queries using PHP

Different ways to run queries (PDO)

// 1. Simple query (no user input) - safe
$stmt = $pdo->query("SELECT * FROM students");

// 2. Prepared statement with positional placeholders (?)
$stmt = $pdo->prepare("SELECT * FROM students WHERE gpa > ? AND faculty = ?");
$stmt->execute([3.5, 'BCT']);

// 3. Named placeholders (:name)
$stmt = $pdo->prepare("UPDATE students SET gpa = :new_gpa WHERE roll = :roll");
$stmt->execute(['new_gpa' => 3.9, 'roll' => '075BCT001']);

// 4. Fetch modes
$all = $stmt->fetchAll(PDO::FETCH_ASSOC);       // array of assoc arrays
$one = $stmt->fetch(PDO::FETCH_ASSOC);          // one row
$col = $stmt->fetchColumn();                    // single value
Never do this (SQL Injection vulnerable):
$pdo->query("SELECT * FROM users WHERE username = '$username'");

6.6 Error Handling and Transaction Management

Error Handling with PDO

$pdo->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);

try {
    $pdo->beginTransaction();

    $stmt1 = $pdo->prepare("UPDATE account SET balance = balance - 1000 WHERE id = 1");
    $stmt1->execute();

    $stmt2 = $pdo->prepare("UPDATE account SET balance = balance + 1000 WHERE id = 2");
    $stmt2->execute();

    $pdo->commit();
    echo "Transfer successful!";
} catch (Exception $e) {
    $pdo->rollBack();
    echo "Transfer failed: " . $e->getMessage();
}

Transaction Management Summary

  • $pdo->beginTransaction() – start transaction
  • $pdo->commit() – save changes
  • $pdo->rollBack() – undo changes on error
  • Automatic rollback if exception thrown before commit
When to use transactions? Any time multiple related queries must succeed or fail together (bank transfer, order processing, student enrollment + fee deduction).
