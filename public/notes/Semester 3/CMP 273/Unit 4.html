Unit 4: Form Handling and Data Validation

4.1 Processing HTML Forms with PHP

PHP processes HTML form data sent via GET or POST methods using superglobal arrays: $_GET, $_POST, and $_REQUEST.

<!-- register.html -->
<form action="register.php" method="post">
    <label>Full Name:</label>
    <input type="text" name="full_name" required><br>

    <label>Email:</label>
    <input type="email" name="email" required><br>

    <label>Password:</label>
    <input type="password" name="password" required><br>

    <button type="submit">Register</button>
</form>
<?php
// register.php
if ($_SERVER["REQUEST_METHOD"] == "POST") {
    $name  = $_POST['full_name'];
    $email = $_POST['email'];
    $pass  = $_POST['password'];

    echo "Registration successful!
";
    echo "Name: $name
Email: $email
";
}
?>
Always check $_SERVER["REQUEST_METHOD"] == "POST" to avoid processing on page refresh (GET).

SuperglobalMethodData VisibilityData LimitBest For
$_GETGETVisible in URL~2048 characters (browser/server limit)Search, pagination, non-sensitive data
$_POSTPOSTHidden in request bodyLarger (php.ini: post_max_size)Forms, login, file upload
$_REQUESTGET + POST + COOKIEDepends on source—General access (not recommended)

// Common $_SERVER useful values
echo $_SERVER['REQUEST_METHOD'];      // POST or GET
echo $_SERVER['REMOTE_ADDR'];         // Client IP
echo $_SERVER['HTTP_USER_AGENT'];     // Browser info
echo $_SERVER['PHP_SELF'];            // Current script path
echo $_SERVER['REQUEST_URI'];         // Full URL after domain

4.3 Preserving User Input

When form validation fails, you want to show the form again with previously entered values preserved.

<form action="" method="post">
    <label>Name:</label>
    <input type="text" name="name" value="<?php echo isset($_POST['name']) ? htmlspecialchars($_POST['name']) : ''; ?>">

    <label>Email:</label>
    <input type="email" name="email" value="<?php echo isset($_POST['email']) ? htmlspecialchars($_POST['email']) : ''; ?>">

    <button type="submit">Submit</button>
</form>
Always use htmlspecialchars() when outputting user input in HTML to prevent XSS attacks.

4.4 Validating and Sanitizing User’s Input

Basic Validation & Sanitization Steps

  1. Check if field is set (isset())
  2. Remove unnecessary characters (trim())
  3. Sanitize input (filter_var(), htmlspecialchars())
  4. Validate format (email, number, length…)
  5. Escape output when displaying
<?php
function cleanInput($data) {
    $data = trim($data);
    $data = stripslashes($data);
    $data = htmlspecialchars($data, ENT_QUOTES, 'UTF-8');
    return $data;
}

function isValidEmail($email) {
    return filter_var($email, FILTER_VALIDATE_EMAIL) !== false;
}

$errors = [];

if ($_SERVER["REQUEST_METHOD"] == "POST") {
    $name = cleanInput($_POST["name"] ?? '');
    $email = cleanInput($_POST["email"] ?? '');

    if (empty($name)) {
        $errors[] = "Name is required";
    } elseif (strlen($name) < 3) {
        $errors[] = "Name must be at least 3 characters";
    }

    if (empty($email)) {
        $errors[] = "Email is required";
    } elseif (!isValidEmail($email)) {
        $errors[] = "Invalid email format";
    }

    if (empty($errors)) {
        echo "Form submitted successfully!";
    } else {
        foreach ($errors as $err) {
            echo "
$err
";
        }
    }
}
?>
Never trust user input!
• Use filter_var() with proper filters
• Avoid mysql_real_escape_string() — use prepared statements instead

4.5 Dealing with Checkbox, Radio Button and List

<form action="" method="post">
    <!-- Radio buttons -->
    <label>Gender:</label>
    <input type="radio" name="gender" value="male" <?php echo (isset($_POST['gender']) && $_POST['gender']=='male')?'checked':''; ?>> Male
    <input type="radio" name="gender" value="female" <?php echo (isset($_POST['gender']) && $_POST['gender']=='female')?'checked':''; ?>> Female<br>

    <!-- Checkboxes -->
    <label>Hobbies:</label>
    <input type="checkbox" name="hobbies[]" value="reading" <?php echo (isset($_POST['hobbies']) && in_array('reading', $_POST['hobbies']))?'checked':''; ?>> Reading
    <input type="checkbox" name="hobbies[]" value="sports" <?php echo (isset($_POST['hobbies']) && in_array('sports', $_POST['hobbies']))?'checked':''; ?>> Sports
    <input type="checkbox" name="hobbies[]" value="music" <?php echo (isset($_POST['hobbies']) && in_array('music', $_POST['hobbies']))?'checked':''; ?>> Music<br>

    <!-- Select (dropdown) -->
    <label>Country:</label>
    <select name="country">
        <option value="np" <?php echo (isset($_POST['country']) && $_POST['country']=='np')?'selected':''; ?>>Nepal</option>
        <option value="in" <?php echo (isset($_POST['country']) && $_POST['country']=='in')?'selected':''; ?>>India</option>
        <option value="us" <?php echo (isset($_POST['country']) && $_POST['country']=='us')?'selected':''; ?>>USA</option>
    </select>

    <button type="submit">Submit</button>
</form>

<?php
if ($_SERVER["REQUEST_METHOD"] == "POST") {
    $gender = $_POST['gender'] ?? 'Not selected';
    $hobbies = $_POST['hobbies'] ?? [];
    $country = $_POST['country'] ?? 'Not selected';

    echo "<p>Gender: $gender</p>";
    echo "<p>Hobbies: " . implode(", ", $hobbies) . "</p>";
    echo "<p>Country: $country</p>";
}
?>
• Checkboxes use name="hobbies[]" → becomes array in $_POST
• Use isset() and in_array() for preserving checked state

4.6 File Upload

<form action="" method="post" enctype="multipart/form-data">
    <input type="file" name="profile_photo" accept="image/*">
    <button type="submit" name="upload">Upload</button>
</form>

<?php
if (isset($_POST['upload'])) {
    $file = $_FILES['profile_photo'];

    if ($file['error'] === UPLOAD_ERR_OK) {
        $uploadDir = "uploads/";
        $fileName = basename($file['name']);
        $targetPath = $uploadDir . time() . "_" . $fileName;

        // Validate file type & size
        $allowed = ['jpg', 'jpeg', 'png', 'gif'];
        $ext = strtolower(pathinfo($fileName, PATHINFO_EXTENSION));

        if (!in_array($ext, $allowed)) {
            echo "Invalid file type!";
        } elseif ($file['size'] > 2 * 1024 * 1024) { // 2MB limit
            echo "File too large!";
        } elseif (move_uploaded_file($file['tmp_name'], $targetPath)) {
            echo "File uploaded successfully: <img src='$targetPath' width='200'>";
        } else {
            echo "Upload failed!";
        }
    } else {
        echo "Upload error: " . $file['error'];
    }
}
?>
Security Checklist for File Upload:
• Always check file type (extension + MIME)
• Restrict file size (php.ini: upload_max_filesize, post_max_size)
• Use unique filename (time + random)
• Store outside web root if possible
• Never execute uploaded files

4.7 Send Email

<?php
if ($_SERVER["REQUEST_METHOD"] == "POST") {
    $to      = "recipient@example.com";
    $subject = "Contact Form Submission";
    $message = "Name: " . $_POST['name'] . "\n";
    $message .= "Email: " . $_POST['email'] . "\n";
    $message .= "Message:\n" . $_POST['message'];

    $headers  = "From: " . $_POST['email'] . "\r\n";
    $headers .= "Reply-To: " . $_POST['email'] . "\r\n";
    $headers .= "Content-Type: text/plain; charset=UTF-8\r\n";

    if (mail($to, $subject, $message, $headers)) {
        echo "Email sent successfully!";
    } else {
        echo "Failed to send email.";
    }
}
?>

<form action="" method="post">
    <input type="text" name="name" placeholder="Your Name" required><br>
    <input type="email" name="email" placeholder="Your Email" required><br>
    <textarea name="message" placeholder="Your Message" required></textarea><br>
    <button type="submit">Send Email</button>
</form>
mail() function limitations:
• Many hosts block or restrict it (spam prevention)
• No HTML support by default
• Better alternatives: PHPMailer, SwiftMailer (SMTP via Gmail, SendGrid, etc.)

Download from GitHub → use SMTP (Gmail, Zoho, SendGrid) → supports HTML, attachments, better error handling.
