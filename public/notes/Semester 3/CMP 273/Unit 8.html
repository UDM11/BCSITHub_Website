Unit 8: PHP Framework

8.1 MVC Model

MVC (Model-View-Controller) is the most popular architectural pattern used in modern PHP web frameworks. It separates application logic into three interconnected components to improve maintainability, scalability, and code organization.

MVC Components Explained

ComponentResponsibilityTypical Files/FoldersHandles
ModelData logic & business rulesapp/Models/, app/Entity/Database operations (CRUD), validation rules, calculations
ViewUser interface / presentationresources/views/, public/assets/HTML, CSS, JavaScript, Blade templates (Laravel)
ControllerHandles user requests & coordinates Model & Viewapp/Http/Controllers/Receives HTTP request, fetches data from Model, passes to View

Typical Request Flow in MVC (Laravel example)

  1. User sends request (e.g., /students/001)
  2. Router matches URL → calls Controller method
  3. Controller asks Model to fetch data (Student::find(1))
  4. Model queries database → returns data
  5. Controller passes data to View
  6. View renders HTML → response sent to browser
Benefits of MVC separation:
• Easier teamwork (frontend & backend developers separate)
• Code reusability (same Model used in API & web)
• Testability (unit test Model & Controller independently)
• Maintainability (change UI without touching logic)

8.2 Benefits of using the PHP Framework

Using a PHP framework (Laravel, CodeIgniter, Symfony, Yii, CakePHP, Slim…) saves time and reduces bugs compared to writing everything from scratch.

Major Benefits (2025 perspective)

BenefitExplanationExample (Laravel)
Rapid DevelopmentBuilt-in tools & conventionsArtisan CLI, Eloquent ORM, Blade templating
Security Built-inProtection against common attacksCSRF protection, input sanitization, bcrypt hashing
Best Practices EnforcedFollows MVC, PSR standardsPSR-4 autoloading, middleware pattern
Database AbstractionWorks with multiple DBs easilyEloquent ORM supports MySQL, PostgreSQL, SQLite, SQL Server
Powerful Features Out-of-BoxAuthentication, routing, validation, caching, queuesLaravel Breeze / Jetstream for auth scaffolding
Community & EcosystemHuge packages (Laravel Nova, Livewire, Filament…)Packagist + Laravel ecosystem
Scalability & PerformanceBuilt-in caching, queue, optimization toolsRedis, Memcached, Horizon, Octane
Testing SupportPHPUnit, Dusk, Pest integratedFeature & unit tests easy
Trade-off: Frameworks add learning curve and overhead.
For very small projects → plain PHP is faster to start.
For medium-large projects → framework saves months of work.

8.3 Getting Started with PHP Framework

Most colleges in Nepal teach **Laravel** as the primary framework (very high job demand). Below is a beginner-friendly guide to get started with Laravel.

Step-by-step: Laravel Installation (2025 method – Laravel 11.x)

  1. Install PHP & Composer
    • PHP ≥ 8.2
    • Composer (dependency manager) → https://getcomposer.org/
  2. Create new Laravel project 
    composer create-project laravel/laravel myproject "11.*"
    or (recommended – Laravel installer) 
    composer global require laravel/installer
laravel new myproject
  3. Go to project folder & start server 
    cd myproject
php artisan serve
    → Open browser: http://127.0.0.1:8000
  4. Basic folder structure (Laravel 11) 
    app/                ← Application core (Models, Controllers, etc.)
bootstrap/          ← Application bootstrap
config/             ← Configuration files
database/           ← Migrations, seeders, factories
public/             ← Publicly accessible (index.php, assets)
resources/          ← Views (Blade), CSS, JS
routes/             ← Route definitions (web.php, api.php)
storage/            ← Logs, file uploads, cache
tests/              ← Automated tests
vendor/             ← Composer packages
  5. First route & controller 
    // routes/web.php
use App\Http\Controllers\WelcomeController;

Route::get('/', [WelcomeController::class, 'index']);

// app/Http/Controllers/WelcomeController.php
namespace App\Http\Controllers;

class WelcomeController extends Controller
{
    public function index()
    {
        return view('welcome', ['name' => 'Umesh']);
    }
}

// resources/views/welcome.blade.php
<h1>Hello, {{ $name }}! Welcome to Laravel.</h1>

Recommended learning path (2025):
1. Learn plain PHP + MySQL CRUD first
2. Understand MVC pattern
3. Start with Laravel (most jobs ask for it)
4. Learn Eloquent ORM, Blade, Routing, Middleware, Authentication
College project tip: Even if syllabus asks for plain PHP, implementing login/registration with classes + MVC structure gives extra marks and looks professional.
