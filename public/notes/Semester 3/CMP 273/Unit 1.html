1.1 Understanding Server-side Scripting and PHP Programming

Server-side scripting refers to programs/scripts that run on the web server (not in the user's browser) to generate dynamic web content, process forms, interact with databases, handle authentication, and perform business logic.

Client-side vs Server-side Scripting

AspectClient-side (e.g. JavaScript, CSS)Server-side (e.g. PHP, Node.js, Python)
Execution locationUser's browserWeb server
Code visibilityVisible in page sourceNot visible to client
Database accessNo direct accessFull access (MySQL, PostgreSQL…)
SecurityLow (can be tampered)High (logic hidden)
Use casesUI interactivity, animationsForm processing, login, file upload, CRUD

What is PHP?

  • PHP = Hypertext Preprocessor (originally Personal Home Page)
  • Created by Rasmus Lerdorf in 1994 → became open-source in 1995
  • Server-side scripting language designed for web development
  • Embedded directly into HTML
  • Very popular: WordPress, Laravel, Drupal, Joomla, Moodle, Facebook (early versions), Wikipedia…
  • Current stable version (2025): PHP 8.3 / 8.4 (with JIT, attributes, typed properties, match expression, nullsafe operator…)
Why learn PHP in 2025?
• Still powers ~77% of websites with known server-side language (W3Techs 2025)
• Huge job market in Nepal & India (WordPress, custom CMS, e-commerce)
• Easy to learn, forgiving syntax, massive community

1.3 Basic Syntax and Data Types

Basic PHP Syntax Rules

  • PHP code starts with <?php and ends with ?>
  • Statements end with semicolon ;
  • Case-sensitive for variables & constants (not for keywords)
  • Comments: // single line, /* multi-line */, # single line
  • Files must have .php extension

1.4 Variables and Constants

Variables

  • Start with $ followed by letter or underscore
  • Case-sensitive: $Age ≠ $age
  • No need to declare type (loosely typed)
  • Can change type dynamically
$firstName = "Hari";      // string
$age = 22;               // int
$age = "twenty two";     // now string (allowed!)

Variable Scope

  • Local: inside function
  • Global: outside function (use global keyword inside function)
  • Static: retains value between function calls
  • Parameter: function arguments

Constants

  • Defined with define("NAME", value) or const NAME = value;
  • Cannot be changed once set
  • Usually uppercase
  • Global scope by default
define("SITE_NAME", "My College Portal");
const PI = 3.14159;

echo SITE_NAME;   // My College Portal
// SITE_NAME = "New Name";   // Error!
