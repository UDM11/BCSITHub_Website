Unit 5: File Handling, Sessions, and Error Handling

5.1 Reading from and Writing to Files

PHP provides several functions to work with files on the server. Always check file permissions and path carefully.

Common File Modes

ModeDescriptionCreates file if not exist?Pointer position
rRead onlyNoBeginning
r+Read & writeNoBeginning
wWrite only (truncates file)YesBeginning
w+Read & write (truncates)YesBeginning
aWrite only (append)YesEnd
a+Read & write (append)YesEnd for write
<?php
// Writing to file
$file = fopen("data.txt", "w") or die("Unable to open file!");
fwrite($file, "Hello Nepal!\n");
fwrite($file, "This is line 2.\n");
fclose($file);

// Reading file
$file = fopen("data.txt", "r") or die("Cannot open file!");
while (!feof($file)) {
    echo fgets($file) . "
";
}
fclose($file);

// One-liner alternatives
file_put_contents("log.txt", "Error occurred at " . date("Y-m-d H:i:s") . "\n", FILE_APPEND);
$content = file_get_contents("data.txt");
echo nl2br($content);
?>
Security note: Never allow user input to directly control file paths (directory traversal attack: ../../etc/passwd)

5.2 Understanding File Permissions and Security Considerations

File Permissions (Linux/Unix style – important for hosting)

PermissionValueMeaningRecommended for PHP files
Readr (4)Can read file—
Writew (2)Can modify file—
Executex (1)Can execute (scripts)—
644rw-r--r--Owner read/write, others readPHP files (.php)
755rwxr-xr-xOwner all, others read/executeFolders, executable scripts
666rw-rw-rw-Everyone read/writeDangerous – avoid!
777rwxrwxrwxEveryone full accessNever use on production!

Security Best Practices for File Handling

  • Never let user input control file path directly
  • Use basename() to strip directory info from uploaded filenames
  • Store sensitive files outside web root (e.g., /var/private/ not /htdocs/)
  • Check file type & size before processing
  • Disable dangerous PHP functions (disable_functions in php.ini)
  • Use open_basedir to restrict file access
  • Log file operations for auditing
777 on uploads folder is very common mistake in student projects → easy to exploit.

5.4 Managing Sessions and Cookies

Cookies (client-side)

// Set cookie (expires in 30 days)
setcookie("username", "ram_sharma", time() + (86400 * 30), "/");

// Read cookie
if (isset($_COOKIE["username"])) {
    echo "Welcome " . $_COOKIE["username"];
}

// Delete cookie
setcookie("username", "", time() - 3600, "/");

Sessions (server-side)

<?php
session_start();   // must be before any output

// Set session variable
$_SESSION["user_id"] = 101;
$_SESSION["role"] = "student";

// Read
echo "User ID: " . $_SESSION["user_id"];

// Unset one variable
unset($_SESSION["role"]);

// Destroy entire session
session_destroy();
?>
Cookie vs Session:
• Cookie: stored in browser, can be tampered, size limit ~4KB
• Session: stored on server, more secure, only session ID in cookie

5.6 Error Handling in PHP: try-catch blocks, exception handling

Basic Error Reporting Control

// development
ini_set('display_errors', 1);
ini_set('display_startup_errors', 1);
error_reporting(E_ALL);

// production
ini_set('display_errors', 0);
error_reporting(E_ALL);  // still log errors

try-catch with Exceptions

<?php
function divide($a, $b) {
    if ($b == 0) {
        throw new Exception("Division by zero!");
    }
    return $a / $b;
}

try {
    echo divide(10, 0);
} catch (Exception $e) {
    echo "Error: " . $e->getMessage();
} finally {
    echo "<br>Operation attempted.";
}
?>

Custom Exception

class ValidationException extends Exception {}

function validateAge($age) {
    if ($age < 18) {
        throw new ValidationException("Age must be 18 or older!");
    }
}

try {
    validateAge(16);
} catch (ValidationException $e) {
    echo "Validation failed: " . $e->getMessage();
} catch (Exception $e) {
    echo "General error: " . $e->getMessage();
}
Best practice in 2025:
• Use exceptions for exceptional cases (not normal flow control)
• Log errors (error_log() or Monolog)
• Use set_error_handler() for custom error handling
• In frameworks (Laravel) → global exception handling is built-in
