4.1 Introduction to SQL

SQL (Structured Query Language) is the standard language for managing and manipulating relational databases. It is declarative (you say what you want, not how to get it).

  • Developed: 1970s at IBM (SEQUEL → SQL)
  • Standardized: ANSI/ISO SQL-86, SQL-92, SQL:1999, SQL:2003, SQL:2011, SQL:2016, SQL:2023
  • Dialects: MySQL, PostgreSQL, Oracle, SQL Server, SQLite, MariaDB, etc.
  • Categories: DDL, DML, DCL, TCL, DQL

SQL is: case-insensitive for keywords (SELECT = select), but case-sensitive for identifiers in some DBMS.

4.4 Structure – Creation, Alteration, Drop, Defining Constraints

-- DDL Examples

-- CREATE TABLE
CREATE TABLE Student (
    sid INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100) NOT NULL,
    email VARCHAR(100) UNIQUE,
    dob DATE,
    gpa DECIMAL(3,2) DEFAULT 0.00 CHECK (gpa BETWEEN 0 AND 4.00),
    dept_id INT,
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (dept_id) REFERENCES Department(dept_id) ON DELETE SET NULL
);

-- ALTER TABLE
ALTER TABLE Student
    ADD COLUMN phone VARCHAR(15),
    MODIFY COLUMN name VARCHAR(120) NOT NULL,
    DROP COLUMN is_active,
    ADD CONSTRAINT chk_gpa CHECK (gpa >= 0);

-- DROP TABLE / TRUNCATE
DROP TABLE Student;
TRUNCATE TABLE TempData;  -- removes data, keeps structure

-- RENAME
RENAME TABLE OldStudents TO Students;

Constraints Summary

  • PRIMARY KEY: NOT NULL + UNIQUE
  • FOREIGN KEY: referential integrity
  • UNIQUE: no duplicates (allows NULL unless combined with NOT NULL)
  • NOT NULL: mandatory value
  • CHECK: custom condition
  • DEFAULT: value if not provided

4.4 Manipulation Commands (INSERT, UPDATE, DELETE, SELECT queries)

-- DML Examples

-- INSERT
INSERT INTO Student (name, email, dob, gpa, dept_id)
VALUES ('Ram Sharma', 'ram@edu.np', '2003-05-12', 3.85, 1);

INSERT INTO Student (name, dept_id)
VALUES ('Sita Gurung', 2);   -- uses defaults for others

-- UPDATE
UPDATE Student
SET gpa = gpa + 0.1,
    email = 'sita.new@edu.np'
WHERE sid = 2 AND gpa < 3.5;

-- DELETE
DELETE FROM Student WHERE dept_id = 5 AND gpa < 2.0;

4.5 Functions, Set Operations, Sub-queries, GROUP BY, HAVING, ORDER BY, LIKE, EXISTS, ANY/ALL, BETWEEN, JOINs, VIEWs

Aggregate Functions

COUNT(*), COUNT(column), SUM(), AVG(), MIN(), MAX()
SELECT dept_id, COUNT(*) AS student_count, AVG(gpa) AS avg_gpa
FROM Student
GROUP BY dept_id
HAVING AVG(gpa) > 3.0
ORDER BY avg_gpa DESC;

Built-in Functions (common examples)

  • String: UPPER(), LOWER(), CONCAT(), SUBSTRING(), LENGTH(), TRIM(), REPLACE()
  • Numeric: ROUND(), CEIL(), FLOOR(), ABS(), POWER(), MOD()
  • Date: NOW(), CURDATE(), DATE_FORMAT(), DATEDIFF(), DATE_ADD(), YEAR(), MONTH()

Subqueries & Correlated Subqueries

-- Scalar subquery
SELECT name, gpa
FROM Student
WHERE gpa > (SELECT AVG(gpa) FROM Student);

-- EXISTS
SELECT name
FROM Department d
WHERE EXISTS (
    SELECT 1 FROM Student s WHERE s.dept_id = d.dept_id
);

-- ANY / ALL
SELECT name
FROM Student
WHERE gpa > ALL (SELECT gpa FROM Student WHERE dept_id = 2);

Views

-- Simple view
CREATE VIEW ActiveStudents AS
SELECT sid, name, gpa, dept_id
FROM Student
WHERE is_active = TRUE;

-- Updatable view (limited conditions)
CREATE VIEW DeptAvgGPA AS
SELECT dept_id, AVG(gpa) AS avg_gpa
FROM Student
GROUP BY dept_id;

-- Materialized view (PostgreSQL / Oracle)
