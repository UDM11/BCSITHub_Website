5.1 Random Variable and its Types

A random variable is a numerical outcome of a random experiment / process. It assigns a real number to each possible outcome in the sample space.

Two Main Types of Random Variables

Type Definition Possible Values Examples (IT related) Probability Representation
Discrete Random Variable Can take only a countable number of distinct values Finite or countably infinite (0,1,2,3,…) • Number of users online at a given time
• Number of failed login attempts in 1 hour
• Number of bugs found in a module
• Number of packets lost in transmission 		Probability Mass Function (PMF): P(X = x)
Continuous Random Variable Can take any value in a given interval Infinite values (any real number in range) • Server response time (in ms)
• Download speed (Mbps)
• CPU utilization percentage
• Time between two consecutive website visits 		Probability Density Function (PDF): f(x)
P(a ≤ X ≤ b) = area under curve
Quick check:
• Can you count the possible values? → Discrete
• Can it take any value in a range (including decimals)? → Continuous

5.2 Expectation and Variance of a Discrete Random Variable

Expected Value / Mean / Expectation (E(X) or μ)

The long-run average value of repetitions of the experiment.

E(X) = μ = Σ [xᵢ · P(X = xᵢ)] (for discrete random variable)

Variance (Var(X) or σ²)

Measures the average squared deviation from the mean — shows spread / risk.

Var(X) = σ² = Σ [(xᵢ – μ)² · P(X = xᵢ)] = E(X²) – [E(X)]²

Standard Deviation

σ = √Var(X) = √σ²
Number of daily website crashes (X):
X | P(X)
0 | 0.4
1 | 0.3
2 | 0.2
3 | 0.1
E(X) = 0·0.4 + 1·0.3 + 2·0.2 + 3·0.1 = 1.0 crash/day
E(X²) = 0²·0.4 + 1²·0.3 + 2²·0.2 + 3²·0.1 = 1.6
Var(X) = 1.6 – (1.0)² = 0.6
σ ≈ 0.7746 crashes/day
In IT: Expectation → average server load, average bugs per sprint
Variance → risk / unpredictability (high variance = unstable system)

5.3 Binomial Probability Distribution

Binomial distribution describes the number of successes in a fixed number of independent Bernoulli trials (yes/no experiments).

Conditions for Binomial Distribution

  1. Fixed number of trials (n)
  2. Each trial has exactly two outcomes (success / failure)
  3. Probability of success (p) is constant for each trial
  4. Trials are independent
P(X = k) = C(n,k) · pᵏ · (1–p)ⁿ⁻ᵏ

Where C(n,k) = n! / (k! · (n–k)! ) = binomial coefficient

Mean, Variance & Standard Deviation

Mean = μ = n·p
Variance = σ² = n·p·(1–p)
Standard deviation = σ = √[n·p·(1–p)]
A web server has 5% chance of failing to respond to a request (p = 0.05). What is the probability that exactly 2 out of 20 requests fail?
n = 20, k = 2, p = 0.05
P(X=2) = C(20,2) × (0.05)² × (0.95)¹⁸ ≈ 0.1887 (18.87%)
Common IT applications:
• Probability that k servers fail out of n
• Number of defective code modules in a release
• Number of successful login attempts vs failed (brute-force detection)

5.4 Poisson Probability Distribution

Poisson distribution models the number of times an event occurs in a fixed interval of time or space when events occur independently and at a constant average rate.

Conditions for Poisson Distribution

  1. Events occur independently
  2. Average rate (λ) is constant
  3. Two events cannot occur at exactly the same instant
  4. Probability of more than one event in very small interval → 0
P(X = k) = (e⁻ˡ · λᵏ) / k!

(λ = average number of events per interval, e ≈ 2.71828)

Mean, Variance & Standard Deviation

Mean = μ = λ
Variance = σ² = λ
Standard deviation = σ = √λ
A web server receives on average 3.2 requests per second (λ = 3.2). Find probability of exactly 5 requests in one second.
P(X=5) = (e⁻³.² × 3.2⁵) / 5! ≈ 0.1088 (10.88%)
Common IT applications:
• Number of server crashes per day
• Number of customer support tickets per hour
• Number of packets arriving at router per second
• Number of bugs reported per week
Poisson is often used to approximate Binomial when n is large and p is small (np = λ remains constant).

5.5 Normal Probability Distribution

The normal distribution (Gaussian / bell curve) is the most important continuous probability distribution in statistics. Many natural and IT phenomena approximate normal distribution when sample size is large (Central Limit Theorem).

Properties of Normal Distribution

  • Symmetric about mean (μ)
  • Mean = Median = Mode = μ
  • Bell-shaped, extends infinitely in both directions
  • Defined by two parameters: mean (μ) and standard deviation (σ)
  • 68-95-99.7 rule:
    • ≈68% within μ ± 1σ
    • ≈95% within μ ± 2σ
    • ≈99.7% within μ ± 3σ
PDF: f(x) = (1 / (σ √(2π))) · e^(-(x-μ)² / (2σ²))

Standard Normal Distribution (Z-distribution)

Normal distribution with μ = 0 and σ = 1

Z = (X – μ) / σ

Z-score tells how many standard deviations a value is from the mean. Use standard normal table (Z-table) to find probabilities.

Server response time is normally distributed with mean 220 ms and SD 35 ms. What is P(response time < 250 ms)?
Z = (250 – 220) / 35 = 0.857
From Z-table: P(Z < 0.86) ≈ 0.8051 → 80.51%
Common IT applications:
• Response time, load time, file size distribution
• Error rates, network latency
• Performance benchmarking
• Anomaly detection (values > μ + 3σ often outliers)
Central Limit Theorem (CLT): Sum / mean of large number of independent random variables tends to normal distribution — even if original variables are not normal.
